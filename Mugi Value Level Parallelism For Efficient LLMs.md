    1、论文工作：
        - 论文要解决的核心问题：已有 Value Level Parallelism（VLP）主要面向大 batch、低精度、激活和权重对称格式的 GEMM，例如 Carat 这类设计；但 LLM 推理并不只是 activation-weight GEMM，还包含 softmax、exp、SiLU/GELU 等非线性操作、weight-only quantization（WOQ）、KV cache quantization（KVQ）、group query attention（GQA）以及小 batch 低延迟推理。已有 VLP 不能直接覆盖这些 LLM 关键路径，常规 systolic/SIMD/vector-array 方案又会在小 batch、非线性单元和硬件面积/能耗上付出较高代价。
        - 论文的主要贡献：第一，将 VLP 推广到 nonlinear approximation，用“value-centric”方式让更重要的输入值获得更高近似精度；第二，为 LLM 中常见的 BF16-INT4 asymmetric small-batch GEMM 重新设计 VLP 映射，使其兼容 WOQ、KVQ 和 GQA；第三，提出 Mugi 架构，把非线性近似和 GEMM 优化放进同一套可复用 VLP array，而不是为 nonlinear 与 GEMM 分别堆独立硬件；第四，通过 RTL、事件级架构模拟器、HuggingFace profiling、45nm synthesis / place-and-route、CACTI7 和碳排模型评估性能、能效、面积与 carbon emission。
        - 论文所处背景：LLM 推理越来越依赖低精度和内存压缩，权重和 KV cache 常被压到 INT4，而 Q/activation 仍常以 BF16 等较宽格式存在；同时在线推理为了满足约 200ms 级别的交互延迟，batch size 通常较小。传统 VLP 的价值复用思想仍然诱人，因为它能把“多个输入共享同一值域或同一 LUT 行”的重复计算变成时间编码与订阅，但 LLM 的非线性函数、asymmetric quantization 和 GQA 使原始 VLP 数据流不再匹配。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：Carat 类 VLP baseline 面向 FP8 对称输入和大 batch GEMM，不能直接处理 softmax/SiLU/GELU；如果把 BF16 activation 直接映射到 Carat 的行方向，7-bit mantissa 会把 temporal signal 拉长到 128 cycles，吞吐会明显下降；如果把 INT4 weight/KV 映射到原 FP8 datapath，又会浪费位宽和硬件。传统 systolic / SIMD / vector array baseline 能做 GEMM 或 nonlinear，但小 batch LLM 容易让大阵列利用率不足，且非线性函数通常需要单独 vector array、Taylor/PWL/PA 近似逻辑或精确 MAC，增加面积、功耗和 embodied carbon。Tensor Core baseline 具有很高吞吐，但论文把它主要放在 NoC/大配置对比中；它不是 Mugi 这种轻量 VLP 架构的直接面积等价替代。
        - 论文的设计方法：Mugi 的核心是把 VLP 重新定义为适配 LLM 的两条路径。非线性路径中，输入按 sign-mantissa-exponent（S-M-E）拆分，mantissa 被四舍五入到 3-bit，exponent 通过滑动 LUT window 只覆盖重要值域；计算分为 input field split、value reuse、mantissa temporal subscription、exponent temporal subscription 四阶段。GEMM 路径中，Mugi 转置 Carat 的映射方式：把 INT4 weight/KV cache 放到行方向，用更窄 datapath；把 BF16 input/Q token 放到列方向，使 token 大维度填满行、GQA group size 8 填满 8 列。架构上用 iSRAM/iFIFO/SW/M-proc/E-proc/TC/PE/PP/oAcc/oFIFO/oSRAM/Vec 等模块，让 nonlinear 与 GEMM 共享主要计算阵列。
        - 方法如何对冲 Baseline 缺陷：对 nonlinear baseline，Mugi 不再用单输入顺序 LUT 或昂贵 MAC 精确计算 exp，而是先从 LUT 读出一整行候选值，再用 temporal subscription 为多个输入并行订阅正确值，从而把 LUT 行复用变成值级并行。对 Carat，Mugi 避免 BF16 mantissa 造成的 128-cycle 行方向时间编码，改由 INT4 weight/KV 在行方向产生短 temporal signal，同时用 BF16 Q/activation 列方向并行填充阵列。对传统 systolic/SIMD，小 batch 下 Mugi 的 8 列配置与 GQA group size 对齐，较少出现大阵列空转；对独立 nonlinear 硬件，Mugi 复用同一 array，减少专用 vector array 和 FIFO 面积。
        - 关键 trade-off：Mugi 接受了近似计算误差、LUT window 配置、离线 profiling、更多专用控制模块和硬件设计复杂度，换取 nonlinear/GEMM 共用阵列、较低乘法器开销、较高小 batch 利用率和更好的能效。其收益依赖 LLM 值分布是否集中、GQA/INT4 是否存在、batch 和矩阵尺寸是否与 8 列 VLP mapping 匹配。论文也承认 layer normalization、RoPE 等操作未被完整支持；MoE 和 multimodal 的泛化主要是推测或部分 workload 证据，完整验证留给未来工作；LUT 目前离线预计算，运行时值分布 drift 需要未来 online adjustment 机制进一步处理。

    3、论文实现：
        - Baseline 如何实现：非线性 baseline 包括精确 vector array、Taylor series、piecewise linear（PWL）和 partial approximation（PA）等硬件近似路径；Taylor 用 Horner's rule 到 9 阶，PWL 使用 22 段并配寄存器/比较器。GEMM baseline 包括 Carat、systolic array（SA）、SIMD array（SD）、FIGNA 扩展、NVIDIA Hopper Tensor Core 抽象和 Mugi-L。论文为了让 Carat 能参与 BF16-INT4 对比，修改其 accumulator 到 BF16，并让其 FP8 datapath 处理 INT4 weight；SA/SD 采用 weight-stationary，Mugi/Carat 使用 output-stationary；Mugi-L 用专用 LUT 做 nonlinear，而不是 Mugi 的 temporal-coding approximation。
        - 新设计如何实现：Mugi 的 nonlinear 端以 iSRAM 作为预计算 LUT，M-proc 处理 sign/mantissa 并把 BF16 mantissa 近似到 3-bit，E-proc 根据 exponent 最大/最小值选择滑动窗口并做 clamp，SW 生成 window，iFIFO 让相邻列错拍进入，TC 把 mantissa 转为 temporal spike，PE 中 AND/OR gate 完成订阅，PP 处理 exponent 订阅和 Zero/INF/NaN 等特殊值。Softmax 先执行 max-subtracted exp，oAcc 同步累加 exp，随后把 sum 写入 oSRAM，再用 Vec array 将 exp 乘以 reciprocal sum 完成归一化。GEMM 端使用 INT4 weight/KV 行映射和 BF16 input/Q 列映射，结合 broadcasting 与 output buffer leaning，把相对 Carat 的总 buffer area 降低 4.5x；多节点时通过 2D mesh NoC、output-stationary dataflow 和 inter-node accumulation 扩展。
        - 实验 / 实现平台：模型包括 Llama 2 7B/13B/70B、Whisper tiny/large、SwinV2 tiny/large 和 ViViT base，其中 Llama 2 70B 支持 GQA group size 8；batch size 覆盖 1-32，LLM profiling 和 loss/perplexity 通过 HuggingFace Transformers 完成，每个模型 profiling 100 次 inference。硬件配置中 Mugi array height 为 32-256、width 为 8，输入字长 16-bit、weight/KV 字长 4-bit，i/w/o SRAM 为 64KB；NoC 采用 4x4 或 8x8，off-chip HBM bandwidth 设为 256GB/s，NoC frequency 400MHz。作者基于 Carat 公开 artifact 扩展事件级架构模拟器，报告 area、leakage power、dynamic energy、cycle count 和 runtime；基础硬件模块用 RTL 实现并在 45nm、400MHz 下综合，memory power 来自 CACTI7；单节点 8x8 Mugi 完整 RTL place-and-route 后面积为 0.056 mm²，频率 408.5MHz，进一步提高综合频率可到 975MHz 但评估仍固定 400MHz 以隔离实现影响。
        - 关键实验设置与指标：非线性准确性用 perplexity/loss、relative error 和 value/exponent distribution 检查，Mugi 与 Taylor/PWL/PA 对比；架构性能用 throughput、energy efficiency、power efficiency、area、latency、operational carbon 和 embodied carbon 衡量。论文报告：在 nonlinear softmax 上，Mugi 相对精确 vector array 达到 45x normalized throughput、481.07x energy efficiency 和 10.69x power efficiency；SiLU 对应 energy efficiency / power efficiency 提升为 667.85x / 14.84x。相对 PWL，softmax 上 throughput、energy efficiency、power efficiency 分别提升 5x、8.53x、1.71x；相对 Taylor softmax 分别提升 10.02x、32.93x、3.28x。端到端 Llama 2 70B GQA、batch 8、seq len 4096 下，Mugi(256) 单节点为 1.39 tokens/s、142.82 tokens/s/nJ、3.37 tokens/s/W；相比 SA(16) 的 0.67、45.97、2.24，对应 2.07x throughput、3.11x energy efficiency、1.50x power efficiency。碳排方面，论文称 Mugi 相比 baseline operational carbon 改善 1.45x、embodied carbon 改善 1.48x。多节点 4x4 Mugi(256) 在表 3 中为 22.19 tokens/s、2314.23 tokens/s/nJ、3.42 tokens/s/W，优于 4x4 SA(16) 的 10.74、770.31、2.35。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：论文没有提出 GPU kernel，而是提出一条面向专用架构的 VLP execution pipeline。最核心的两类路径是 nonlinear VLP approximation pipeline 和 asymmetric small-batch GEMM VLP pipeline；二者共享 Mugi array。nonlinear pipeline 可以概括为“拆输入字段 -> 读取并滑动 LUT window -> mantissa temporal subscription -> exponent temporal subscription -> softmax 累加/归一化”；GEMM pipeline 可以概括为“INT4 weight/KV 行方向时间编码 -> BF16 activation/Q 列方向并行 -> output-stationary accumulation -> 可选 NoC 跨节点累加”。
        - 新 pipeline/kernel 的执行流例子：以一个 softmax 输入向量为例，Mugi 先做常规 max-subtraction，然后每个输入进入 M-proc/E-proc。M-proc 把 BF16 的 sign 和 mantissa 拆出，并把 mantissa 近似成 3-bit；E-proc 观察当前 mapping 中的 exponent 范围，选择一个大小为 8 的滑动 LUT window，并处理 underflow/overflow。随后 iSRAM 按 mantissa 顺序逐周期读出一整行预计算 exp 结果，SW 取出当前窗口，iFIFO 将相邻列错开一个 cycle，保证列间流水。TC 在计数器值等于近似 mantissa 时产生 temporal spike，PE 用这个 spike 订阅对应 LUT row；PP 再用 exponent temporal spike 从该 row 的 window 中选择最终 exp 值，并处理 Zero/INF/NaN。对于 softmax，oAcc 在 exp 产生时同步累加所有 exp，oFIFO/oSRAM 保存 sum，最后 Vec array 用 reciprocal sum 乘回每个 exp，得到 softmax 输出。
        - 新 pipeline/kernel 的执行流例子：以一个 BF16-INT4 attention/FFN GEMM tile 为例，INT4 weight 或 INT4 KV cache 被映射到 Mugi 的行方向，因为 4-bit 值只需要短 temporal signal；BF16 activation 或 Q token 被映射到列方向，避免宽 mantissa 拉长行方向时间编码。输入 token 的大隐藏维度用于填满行，GQA 中 8 个 Q token 共享 KV 的结构用于填满 8 列。PE 不走传统 MAC，而是用 temporal spike 和 value reuse 选择/累加对应乘积，输出在 oAcc 中以 output-stationary 方式累加；若使用多节点，GEMM 被均匀 tile 到 2D mesh NoC 上，各节点输出局部累加结果，再做 inter-node accumulation。这样，Mugi 把原本“非线性单元 + GEMM array + 大量 MAC/独立 FIFO”的路径压缩成一套共享 VLP array 上的时间编码、订阅和累加流程。
