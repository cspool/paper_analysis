# Markdown Split by ## Headings

## Parameters

调用本 skill 时，用户**必须**提供以下两个路径参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `{{src_md_path}}` | 源 Markdown 文件路径 | `/data3/paper_analysis/repo_2026/knowledge_repo/知识库_算法pipeline.md` |
| `{{dst_dir_path}}` | 拆分后的 md 文件输出目录 | `/data3/paper_analysis/knowledge_notes/算法知识笔记` |

## Overview

读取 `{{src_md_path}}`，以 `##` 标题作为分隔符将内容拆分。每个 `##` 标题下的内容写入以该标题命名的独立 md 文件中，保存到 `{{dst_dir_path}}`。第一个 `##` 之前的内容忽略。

## Workflow

### 第一步：读取源文件

读取 `{{src_md_path}}` 的完整内容。

### 第二步：按 ## 拆分内容

1. 以行为单位扫描内容，识别所有以 `## ` 开头的行（即 level-2 Markdown 标题）。
2. 忽略第一个 `##` 标题之前的所有内容。
3. 对于每个 `##` 标题行，将标题文本（去除 `## ` 前缀）作为该段的起点，直到下一个 `##` 标题行（或文件末尾）之间的所有内容作为该段的内容。
4. 如果内容中存在更深层级的标题（如 `###`、`####`），保留原样作为该段内容的一部分。

### 第三步：生成输出文件

1. 确保 `{{dst_dir_path}}` 目录存在，不存在则创建。
2. 对于每个 `##` 段，以标题文本作为文件名生成 `.md` 文件。文件名生成规则：
   - 取标题文本，去除首尾空白
   - 将文件名中的非法字符（`/`、`\`、`:`、`*`、`?`、`"`、`<`、`>`、`|`）替换为 `_`
   - 文件名过长时适当截断（建议不超过 100 字符）
3. 每个文件内容为：以原 `## 标题` 行开头，后接该段的内容。
4. 如果同名文件已存在则覆盖。

### 第四步：输出报告

列出：
- 源文件路径
- 识别到的 `##` 标题数量
- 生成的每个文件路径
- 任何跳过的内容或异常
