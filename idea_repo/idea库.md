## Towards High-Goodput LLM Serving with Prefill-decode Multiplexing

- baseline方法是什么？
  Baseline 有两类。第一类是 Splitwise / DistServe / SGLang-PD 式 prefill-decode disaggregation：系统框架把 prefill 和 decode 放到不同实例或 GPU 组上，decode 实例为了 TBT SLO 保留固定资源，prefill 实例处理新请求；kernel 层仍按各实例内部的普通 prefill / decode execution 发射；硬件层的 GPU 计算资源和显存 / KV cache pool 被实例边界一起切开。这个做法在负载波动时容易让某些 GPU 空闲，并因为每个实例维护独立或更小的 KV cache pool 降低 cache hit rate，论文例子中 cache hit rate 可从 36.6% 降到 4.2%。LoongServe 虽可动态调 GPU 资源，但会释放原 GPU 上的 KV cache，导致多轮请求重算。第二类是 chunked-prefill / SARATHI-Serve / SGLang chunked-prefill：系统框架把 prefill 切成 chunk 并与 decode iteration 放入同一个 token budget；kernel调度上 prefill chunk 和 decode batch 绑定执行，SGLang baseline 使用 FlashInfer 融合 prefill / decode attention kernel；硬件层仍在同一 GPU 上顺序或融合执行。缺陷是 token budget 同时控制 decode SLO 和 GPU 饱和，Llama3-70B 在 8 张 A100 上约需 4K token budget 才能饱和 GPU，但 100ms TBT SLO 下合规预算约 256；长 reused context 还会让 prefill chunk 反复访问历史 KV cache，恶化 TBT。模型推理算法层没有改变模型结构；编译框架层论文未明确说明。

- 论文方法是什么？如何对应解决Baseline的缺陷？
  MuxWise 的方法是在同一 GPU、同一进程内做 prefill-decode multiplexing，只切分计算 SM，不切分显存和 KV cache pool。系统框架上，SGLang 被扩展为 MuxWise：bubble-less multiplex engine 将 prefill 拆成 layer-wise execution，contention-tolerant estimator 用 offline profiling 得到 solo-run predictor 和 decode contention guard，SLO-aware dispatcher 优先满足 decode TBT，再把剩余 SM 给 prefill，并允许短 prefill 非递归抢占长 prefill。kernel / runtime 层上，GreenContext 将 prefill / decode CUDA streams 绑定到不同 SM 集合；decode 以低开销 graph-level 路径优先发射，prefill 按 transformer layer units 发射，CUDA event polling 用于 query-based synchronization。硬件层上，同一组 A100 / H100 / H200 GPU 的 SM 被动态重分区，而 HBM、模型权重和 KV cache pool 保持共享。一个请求执行例子是：decode batch 先由 estimator 判断 60% SM 已能满足 TBT SLO，于是调度器把 60% SM 给 decode，把 40% SM 给一个 prefill batch 的若干层；decode 和 prefill streams 并发执行，prefill 最后一层完成后通过 event polling 合并进后续 decode batch；若短 prefill 请求即将错过 TTFT，调度器可暂停长 prefill 的后续 layer，先执行短请求。这样相对 disaggregation 避免固定实例空闲和 KV cache pool 缩小，相对 LoongServe 避免动态迁移后丢失跨请求 KV 复用，相对 chunked-prefill 解耦 token budget 与 decode SLO，使 decode 小延迟预算下仍可用剩余 SM 推进 prefill。模型推理算法层不改模型；编译框架层论文未明确说明。
