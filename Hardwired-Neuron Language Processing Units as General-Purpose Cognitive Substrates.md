    1、论文工作：
        - 论文要解决的核心问题：LLM inference 的能耗和成本瓶颈本质上来自大规模权重参数被当作“动态数据”反复从存储层级读取。GPU、NPU 以及 Groq LPU / Cerebras WSE 这类 LPU 虽然把架构向 LLM 推理专用化，但仍然保留较强可编程性或把模型权重放在 SRAM / DRAM / HBM 中，在 autoregressive decoding 的每一步仍有巨大的参数读取与控制开销。论文进一步提出一个更激进的问题：既然生产服务通常长期部署一个或少数主力 LLM，能否把一个 LLM 的权重直接硬连线进处理器结构，让自然语言 prompt 成为高层“指令流”，从而用极端专用化换取数量级能效。
        - 论文的主要贡献：第一，提出 Hardwired LPU / HNLPU 这一完全硬连线 gpt-oss 120B FP4 模型的处理器范式，把权重参数物理嵌入计算结构，消除权重 fetch。第二，提出 Metal-Embedding，使用高层金属线的 3D 拓扑来表达 FP4 权重，而不是用二维硅器件 cell 阵列保存权重；论文称相对 Cell-Embedding 获得约 15x 密度提升，并让 60/70 层 photomask 在不同芯片间同质共享。第三，提出 Sea-of-Neurons 金属可编程 structured ASIC，把参数无关的 Hardwired-Neuron 阵列预制出来，只在 M8-M11 金属层写入模型权重，从而把 photomask 成本降低约 112x。第四，给出一个 16 芯片、4x4 row-column 互连、直接实现 gpt-oss 120B 推理流程的 HNLPU 系统和六级 pipeline 数据流，并用 RTL、后端布局和系统建模估计性能、功耗、面积、TCO 与碳排。
        - 论文所处背景：该工作处在 LLM 专用处理器、structured ASIC、bit-serial arithmetic、model-specific dataflow 和可持续 AI 基础设施的交叉点。传统硬连线神经网络由于任务和模型更新太快而缺乏实用价值；LLM 的出现改变了这一假设，因为一个大模型可以作为通用认知 substrate 服务大量任务。不过现代 LLM 的参数规模又让朴素硬连线不可行：论文估算直接用 CMAC 阵列硬连线 gpt-oss 120B 需要约 176,000 mm2、200+ 个芯片和 200+ 套异质 photomask，photomask NRE 可达约 6B 美元，远超可接受范围。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：GPU/H100 baseline 的瓶颈是权重和 KV/activation 访问带来的 memory hierarchy 能耗，以及 instruction-driven 执行带来的调度、解码和控制开销；即使用 TensorRT-LLM 优化，同一 gpt-oss 120B 在论文测量中吞吐只有 45 tokens/s，远低于 HNLPU。WSE-3 这类 wafer-scale LPU 已经更接近模型专用推理，但仍把权重作为预加载数据而非物理常量，论文通过公开 cloud service 测得 2,940 tokens/s，并用公开功耗数据估算能效。朴素 hardwired baseline / Cell-Embedding 的问题不是执行效率，而是物理面积和 NRE：FP4 constant multiplier 虽比通用 multiplier 小，但每个 neuron 仍需大量 constant multipliers 和宽加法树，导致芯片数量、异质 mask 数量和重制成本爆炸。MAC Array baseline 则需要 SRAM 保存权重并反复读取，执行周期和能耗都受限于存储访问。
        - 论文的设计方法：HNLPU 的底层单元是 Hardwired-Neuron。它先利用 FP4 权重只有 16 个取值这一事实，用 distributive law 把多个相同权重对应的输入先累加，再乘以该权重值，形成 accumulate-multiply-accumulate，而不是对每个输入做独立 multiply-accumulate。随后用 bit-serialization 和 Carry Save Adder 树把输入从 LSB 到 MSB 串行处理，用时间换面积。最后把“某个输入乘以哪个 FP4 常量”编码为“该输入连到哪个 accumulator 区域”的金属线拓扑，也就是 Metal-Embedding。Sea-of-Neurons 则把 HN 的硅器件和低层局部金属做成参数无关阵列，所有芯片共享 FEOL/M0-M7 以及 M12+ 等同质 mask，只在 M8-M11 的 10 层 DUV mask 中写入各芯片不同权重。
        - 方法如何对冲 Baseline 缺陷：相对 GPU / WSE，HNLPU 把权重从运行期数据变成物理结构，直接消除 weight fetch、指令调度和通用计算阵列复用开销；每层权重都有自己的 HN 资源，因此 36 个 transformer layer 可以天然形成 inter-layer pipeline。相对 Cell-Embedding，Metal-Embedding 不再用大量硅器件保存每个权重，而是用金属连线把输入路由到 16 个权重值区域，硅器件参数无关，面积从 CE 的 14.3x SRAM baseline 降到 ME 的 0.95x SRAM baseline。相对朴素 hardwired LPU，Sea-of-Neurons 让 16 个芯片共享绝大多数 mask，初始 tapeout 的估计 NRE 降到 59.46M-123.5M 美元量级，参数更新 re-spin 降到 18.53M-37.06M 美元量级。
        - 关键 trade-off：HNLPU 接受极低模型灵活性和 re-spin 更新成本，换取极高推理吞吐和能效；它适合长期、高容量、少数主力模型的 cloud serving，不适合频繁实验、训练、模型结构快速变化或小规模部署。Metal-Embedding 也接受 bit-serial 带来的单个 HN 速度折中，但靠全模型、全层、全芯片并行弥补。系统层面，16 个大芯片和 CXL 3.0 row-column 互连降低了权重访问，但把瓶颈转移到 inter-chip collective communication，论文的执行时间分解显示短上下文下 CXL communication 是主要成分，长上下文下 attention 计算逐步占主导。论文还依赖 5 nm PDK、P&R、routing density、热密度和 yield 假设；这些是高可信物理估计，但不是流片实测芯片。

    3、论文实现：
        - Baseline 如何实现：Embedding methodology 对比中使用三个 5 nm 设计：MAC Array，即 64 KB SRAM 加 1024 个传统 MAC 的计算阵列；Cell-Embedding，即用 constant multiplier / CMAC cell 保存权重；Metal-Embedding，即论文的 HN 方案。系统级 baseline 包括 NVIDIA H100 和 Cerebras WSE-3。H100 baseline 在一台 80 GB H100、3.35 TB/s 带宽的服务器上直接测量，模型通过 TensorRT-LLM 部署，结果取多次平均。WSE-3 baseline 的吞吐通过公开 Cerebras cloud service 在 gpt-oss 120B 上测量，功耗无法直接从 cloud 得到，因此采用公开报告中的系统功耗。HNLPU 使用 post-PnR simulation 捕捉寄生参数和 wire delay，并基于确定性的 Token-In-Token-Out 执行模型估算。
        - 新设计如何实现：作者用 Verilog RTL 实现了 HN Array、Control Unit、VEX、Interconnection Engine 和片上 Attention Buffer，并用 extensive test cases 验证 RTL 正确性。物理实现采用 Synopsys Design Compiler 综合、Synopsys IC Compiler 布局布线，目标是 5 nm technology；PrimeTime PX 使用 workload-derived SAIF switching activity 估算静态和动态功耗；片上 SRAM 用同节点 Memory Compiler 生成分析。单芯片包括 HN Array、VEX、Control Unit、Attention Buffer、Interconnect Engine 和 HBM PHY；每片总面积 827.08 mm2、功耗 308.39 W，其中 HN Array 占 573.16 mm2，Attention Buffer 占 136.11 mm2。全系统 16 芯片，总 die area 13,232 mm2，总系统功耗约 6.9 kW。
        - 实验 / 实现平台：模型是 OpenAI gpt-oss 120B 的 4-bit quantized MoE 版本，hidden size 2880、36 层、128 experts、每次激活 top-4 experts，论文将其权重硬连线进 HN Array。系统互连是 4x4 row-column fully-connected fabric，芯片间使用 CXL 3.0 over PCIe PHY，论文按 <100 ns latency 和 128 GB/s per x16 link 建模。每个 compute module 采用 2.5D packaging，配独立 HBM 存 embedding table 和 KV Cache；片上 320 MB Attention Buffer 由 20,000 个 16 KB bank 组成，1W1R、32-bit access width，用于 KV Cache 和 residual activation，超过片上容量时才 offload 到 HBM。多芯片通信延迟和功耗用 CNSim 建模，单芯片性能用作者自建 cycle-level simulator。
        - 关键实验设置与指标：layout feasibility 评估在 SSG、0.675 V、125C worst-case 条件下完成 timing closure 到 1.0 GHz；ME 层 M8-M11 routing density 低于 70%，layout DRC/LVS clean，寄生参数平均 R=164 ohm、C=7.8 fF，平均/峰值功率密度约 0.3/1.4 W/mm2。算子级结果显示 CE/SRAM(MA)/ME 面积比为 14.3x/1x/0.95x，ME 在 execution cycles 和 energy 上优于 MA 与 CE，因为它既全并行又避免 SRAM 访问和 CE 的大面积 leakage。系统级结果显示 HNLPU 在 gpt-oss 120B、2K token length 下达到 249,960 tokens/s，H100 为 45 tokens/s，WSE-3 为 2,940 tokens/s；能效为 36,226 tokens/kJ，H100 为 34.6 tokens/kJ，WSE-3 为 127.8 tokens/kJ；面积效率为 18.89 tokens/(s*mm2)，远高于 H100 的 0.055 和 WSE-3 的 0.064。TCO 分析中，低容量单 HNLPU 场景的 3-year dynamic TCO 为 96.62M-197.8M 美元，对比等吞吐 H100 cluster 的 191.2M 美元；OpenAI-scale 高容量场景为 118.9M-229.4M 美元，对比 H100 的 9,563M 美元。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：论文没有提出传统意义上的 GPU fused kernel，而是提出 HNLPU Token-In-Token-Out hardware inference pipeline。它由三层机制叠加：第一是 Metal-Embedding HN Array，把所有固定权重计算变成硬连线常量数据流；第二是 16-chip row-column dataflow，把 GQA、attention、MoE FFN 映射到列组、行组和全芯片 collective；第三是 36-layer inter-layer pipeline 加每层 six-stage intra-layer pipeline。由于每层权重都有独立 HN 资源，HNLPU 可以让不同 token / sequence 同时处在不同 transformer layer 和不同 layer 内 stage 中，理论最大并发为 6 x 36 = 216 个请求或 sequence。
        - 新 pipeline/kernel 的执行流例子：一个 token index 输入 HNLPU 后，首先从 HBM 查 embedding dictionary，得到形状为 (1, 2880) 的 token activation。该 activation 进入 36 个 transformer blocks。对 GQA 的 Q/K/V projection，权重按列组分布：每个 chip 取输入的 (1, 720) slice，与本地硬连线的 Wq/Wk/Wv slice 相乘产生 partial sums；Q 通过 column-wise all-reduce 得到 16 heads 的 query，K/V 根据 token position len mod 4 放到对应 chip 的 KV-cache 中，形成后续 attention 访问所需的局部布局。
        - 新 pipeline/kernel 的执行流例子：attention score 阶段，每个 column group 内 chip 都持有完整 query heads，而 K/V cache 按 sequence 维度切片保存在各自 Attention Buffer 中。VEX 读取本地 K slice，计算 Q*K 的局部 attention weight；column-wise all-reduce 后，每个 chip 对本地 fragment 做 softmax normalization。随后 VEX 读取本地 V slice，执行 Softmax(Q*K)*V，得到 partial attention output，再通过 column-wise all-reduce 汇总为该列组的 attention heads。输出 projection Wo 按 row-wise partition：各 chip 产生 (1,720) partial Xo，经 row-wise all-reduce 和 column-wise all-gather 后得到所有 chip 上完整的 (1,2880) Xo。
        - 新 pipeline/kernel 的执行流例子：MoE FFN 阶段，所有 chip 都有完整 Xo。系统先执行 RMSNorm，再用复制到所有芯片上的 Wrout 计算 router 分数；因为 router 权重只占约 0.01% 总权重，复制它可以避免通信。top-k 和 softmax 选出 4/128 experts 后，生成 masked input tensor。128 个 experts 均分到 16 个 chip，每片负责 8 个 experts；各 chip 本地完成 Wup 和 Wgate projection，执行 SwiGLU，再做 Wdown projection。最后所有 chip 对 expert outputs 和 expert weights 做 all-chip all-reduce，得到 FFN 输出 Y，并与 residual 结果组合为下一层输入。
        - 新 pipeline/kernel 的执行流例子：six-stage intra-layer pipeline 可概括为：Stage 1 接收上一层的 Y partial-sum / all-chip all-reduce 结果；Stage 2 在 HN-Q/K/V 中做 Q/K/V projection 与 column group all-reduce；Stage 3 在 VEX 上做 attention；Stage 4 做 Softmax(Q*K)*V 的 column group all-reduce；Stage 5 做 HN-Xo、row group all-reduce 和 all-gather；Stage 6 做 RMSNorm、HN-Router、top-k、HN-UP/GT、SwiGLU、HN-DOWN 和 all-chip all-reduce。prefill 阶段同一序列内 token 无自回归依赖，可以像流水线一样填入；decode 阶段同一 sequence 依赖前一 token，但不同 sequence 相互独立，因此通过 Continuous Batching 在 216 个 slot 中动态补入新 sequence，保持 pipeline 饱和。这个 pipeline 的目标不是改变 Transformer 数学，而是把权重固定性、层间独立硬件资源和 batch scheduling 结合起来，使几乎所有面积、功耗和时间都用于有效推理计算。
