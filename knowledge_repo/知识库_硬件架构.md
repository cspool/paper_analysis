## Streaming Multiprocessor（SM）

术语解释：
SM 是 NVIDIA GPU 上承载并行计算的主要执行资源。论文把 SM 作为 MuxWise 动态计算分区的基本资源单位：prefill 和 decode 不是被拆到不同 GPU 或实例，而是在同一 GPU 内获得不同数量的 SM。

术语关联术语的使用例子：
MuxWise 通过 GreenContext 把 CUDA streams 绑定到不同 SM 集合，并以 16 SM 作为 coarse-grained partition 粒度进行 profiling 和调度。论文解释 16 SM 粒度也与 H100 及更新 GPU 的 thread block cluster 等特性有关。

涉及论文标题：
- Towards High-Goodput LLM Serving with Prefill-decode Multiplexing

## Memory Bandwidth Contention

术语解释：
Memory bandwidth contention 是空间并发执行时多个阶段争用 HBM / memory subsystem 带宽造成的干扰。论文指出现有 GPU spatial partition 技术主要能控制 SM 计算资源，却难以精确控制共享 memory bandwidth，因此 decode latency 可能出现不可预测 slowdown。

术语关联术语的使用例子：
MuxWise 的 contention analysis 显示，在 prefill 和 decode multiplexing 下，decode slowdown 可从接近 0 到约 30% 不等。系统因此用 contention guard 对 decode latency 做 worst-case estimation，再由 SLO-aware dispatcher 选择 SM partition。

涉及论文标题：
- Towards High-Goodput LLM Serving with Prefill-decode Multiplexing

## CUDA MIG / CUDA MPS

术语解释：
CUDA MIG 和 CUDA MPS 是论文讨论的 inter-process GPU spatial sharing 技术。它们可以在不同进程或实例间共享 GPU，但论文认为它们不适合 MuxWise 的需求，因为计算资源调整不够灵活，并且跨进程 prefill / decode 会引入通信与内存共享问题。

术语关联术语的使用例子：
MuxWise 对比 MIG、MPS 和 GreenContext 后选择 GreenContext：MIG / MPS 难以低开销动态重配 SM，而 GreenContext 在同一进程内调整 stream 到 SM 的绑定，使 prefill 和 decode 可以共享同一个 KV cache pool。

涉及论文标题：
- Towards High-Goodput LLM Serving with Prefill-decode Multiplexing

## NVLink

术语解释：
NVLink 是 NVIDIA 多 GPU 服务器中的高速 GPU 间互连。论文主实验平台为 8 张 A100-80GB GPU，GPU 之间通过 NVLink 连接并提供 600 GB/s 带宽；该术语在论文中主要作为实验硬件环境，而不是 MuxWise 的核心机制。

术语关联术语的使用例子：
Llama-70B 等模型在 8 GPU tensor parallelism 配置下运行，底层需要 GPU 间通信。MuxWise 的核心优化发生在每张 GPU 内的 SM partition 与同进程显存共享，但实验平台仍依赖 NVLink 支撑多 GPU 部署。

涉及论文标题：
- Towards High-Goodput LLM Serving with Prefill-decode Multiplexing
