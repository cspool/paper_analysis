    1、论文工作：
        - 论文要解决的核心问题：现有 PIM LLM 推理加速器主要解决 decoding attention 的内存带宽瓶颈，却默认完整 KV cache 能放进 HBM-PIM。长上下文和长输出会使 KV cache 线性增长到数十 GB 甚至数百 GB，而 bank-level PIM 本身还会牺牲一部分 DRAM density，形成论文称为 PIM capacity wall 的容量墙。若把溢出的 KV cache offload 到 CPU/host memory，PCIe/CPU 访存带宽会吞掉 PIM 的性能收益；若使用 sparse attention，散乱访问又破坏 PIM 需要的连续性和局部性；若直接把传统量化/反量化硬件塞进 PIM，则 INT/scale/control 逻辑带来不可接受的面积开销。
        - 论文的主要贡献：AQPIM 提出面向 PIM 的 activation/KV cache 量化与 attention 计算协同设计。它用 Product Quantization(PQ) 在线压缩 KV cache，把 KV 表示成 codebook 与 index，并在 PIM 内直接用 codebook/index 计算近似 attention，避免恢复完整 KV。为解决 PQ 在 attention 上的精度和访存问题，论文提出 importance-weighted k-means、channel pre-sorting、page-aware windowed clustering、intra-row indirection，以及 BankPE/BufferPE 分工的数据流。评测中，AQPIM 在约 80% KV memory reduction 前保持接近 PQCache 的准确率趋势，并相对理想化容量无限 AttAcc! 获得 3.4x speedup；相对 GPU baseline，decoding energy efficiency 最高提升 14.29x。
        - 论文所处背景：LLM 长上下文推理的 decoding 阶段每生成一个 token 都要访问历史 KV cache，注意力中的 GEMV 高度 memory-bound。AttAcc!、NeuPIMs 等 PIM 方案利用 HBM 内部带宽做 attention，但容量随上下文扩展变成新瓶颈。传统 KV 缓解路线包括 eviction、offloading/sparsification、uniform/non-uniform quantization；其中 eviction 会不可逆丢 token，offloading 受外部带宽和随机访问限制，普通量化对 PIM 不友好。AQPIM 的切入点是：activation/KV 分布具有上下文相关的 locality/similarity，PQ 能利用这种结构；PQ 的在线聚类带宽需求在 GPU/CPU 上很重，但正好可以消耗 PIM 未充分利用的内部带宽。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：架构 baseline 包括 GPU+HBM 和 AttAcc! 式 GPU+HBM-PIM。GPU+HBM 在 KV cache 超出 GPU memory 后必须经 CPU offload，通信可占 decoding latency 的 90% 到 98.5%；AttAcc! 能加速 memory-bound attention，但不解决 KV cache 放不进 PIM 的容量问题，论文指出其甚至可能需要大量 HBM-PIM device 支撑较短上下文。算法 baseline 包括 SnapKV、PQCache、SKVQ。SnapKV 通过保留重要 token 降低容量和计算量，但 token eviction 会损失信息；PQCache 用 PQ 做检索但仍保留完整 KV 并从 CPU 取回原始 KV，不能摆脱 host bandwidth；SKVQ 等量化方法可降低容量，但在 GPU 上常需要 upcast，且传统量化若搬到 PIM 需要额外 scale/INT 单元。
        - 论文的设计方法：AQPIM 把 KV cache 表示为 per-head PQ codebook 和 indices。prefilling 阶段 GPU 生成 QKV 并计算正常 attention/projection/FFN，PIM 并行接收 KV、执行 k-means，生成 key/value codebook 与 index；decoding 阶段 GPU 只把新的 qkv 发送给 PIM，PIM 追加新 token 的 index，并用 compressed format 直接生成 attention output 再回传 GPU。算法上，importance-weighted k-means 用最近 t=32 个 token 的 attention score 求权重，使高注意力 token 在聚类中量化误差更小；channel pre-sorting 用离线 calibration dataset 根据 cosine similarity 把相关 channel 分到同一 subvector，并把排序矩阵吸收到 Wq/Wk/Wv/Wo 中，避免 runtime overhead。硬件上，AQPIM 使用 BankPE 做 ADD/MUL/SUM 密集的 distance calculation、centroid calculation、ATNK/ATNV，用 BufferPE 做 cluster assignment、softmax、DIV/EXP 等数据汇聚或大面积操作。
        - 方法如何对冲 Baseline 缺陷：相对 GPU/CPU offload，AQPIM 不把完整 KV 留在 CPU，也不在每步 decoding 取回原始 KV，而是在 PIM 内维护 codebook/index，降低容量和跨 PCIe 通信。相对 AttAcc!，AQPIM 不只是提高内部带宽，而是减少 KV cache footprint，使 PIM 容量随长上下文更可扩展。相对普通量化，AQPIM 不在 BankPE 增加复杂 INT/dequantization pipeline，而是把 attention 的 qK^T 转换为 query subvector 与 key codebook 的小规模乘法，再按 key index lookup/sum；value 侧也用 value codebook/index 重构或近似参与 ATNV。相对 naive PQ，page-aware windowed clustering 把 lookup 限定在 DRAM row 内，intra-row indirection 让随机逻辑访问变成一次 row activation 后的 column decoder 选择，从而避免频繁 DRAM row activation。
        - 关键 trade-off：第一，AQPIM 接受近似 attention 和 PQ 压缩带来的精度风险，因此必须保留 sink tokens、recent sliding window tokens，并引入 weighted clustering/pre-sorting；论文显示在 aggressive compression 下这些优化对准确率很关键。第二，PQ codebook generation 需要在线聚类，AQPIM 依赖 prefilling 阶段 PIM 与 GPU 并行来隐藏开销，并把 k-means 固定为 4 次迭代；这是一种吞吐/精度/隐藏开销的折中。第三，intra-row indirection 要求 centroid/window 设计适配 row buffer，例如 1KB row buffer 存 512 个 FP16 partial products，因此默认 512 centroids 与 page-aware window 的硬件约束绑定。第四，AQPIM 的收益集中在 attention/KV 路径，projection/FFN 等非加速部分仍限制 end-to-end speedup。

    3、论文实现：
        - Baseline 如何实现：架构评测比较 GPU+HBMs、GPU+HBM-PIMs(AttAcc!)、GPU+AQPIM。常规 GPU baseline 使用 1 个 NVIDIA H100 GPU core 和 5 个 16GB HBM modules；PIM-enabled 系统将其中 4 个 16GB HBM 替换为 4 个 16GB HBM-PIM，用于 KV cache storage，其余 HBM 存模型参数等数据；CPU offload 实验使用 Intel Xeon Platinum 8480+。算法 baseline 中 PQCache、SKVQ、SnapKV 均在 GPU 上运行，因为它们本身不是为 PIM 设计；其他 compression method 的 memory reduction ratio 设为 80%。
        - 新设计如何实现：AQPIM 的 PIM command 在 AttAcc! 设计上扩展，包括 PIM_SET_CONFIG、PIM_MAC_AB、PIM_SFM、PIM_RET、PIM_MV_BA、PIM_MV_BF、PIM_RD、PIM_WR、PIM_ACT_AB 等。BankPE 放在 DRAM bank 附近，使用已有 FP16 MAC/ADD/MUL/SUM 类能力，不增加专门量化 ALU；BufferPE 位于 HBM buffer die，处理 MIN、DIV、EXP、softmax 和跨 bank 汇聚。intra-row indirection 复用 1KB row buffer 与 column decoder，只增加从 GRF index 到 column decoder 的选择/重定向逻辑。数据映射上，每个 attention head 映射到单独 HBM，每个 head 内的 subvector set 映射到不同 bank，以提高 BankPE utilization 并避免 inter-HBM/inter-bank transfer。
        - 实验 / 实现平台：准确率实验使用 Mistral-7B-Instruct-v0.2 和 Llama-3.2-3B-Instruct，二者上下文窗口分别为 32K 和 128K，模型推理采用 bfloat16。任务使用 LongBench，覆盖 single/multi-document QA、summarization、few-shot learning、synthetic tasks、code completion 等类别，并选代表任务做整体趋势评估。系统评测使用定制 GPU-PIM simulator，该 simulator 基于 AttAcc! simulator，而 AttAcc! simulator 又是面向 LLM inference 修改的 Ramulator/Ramulator2 系列。仿真输入为 system configuration、model details、input configurations，输出 execution time 与 energy consumption。
        - 关键实验设置与指标：AQPIM 默认 subvectors m=32、centroids K=512；保留前 8 个 sink tokens 和最近 32 个 sliding window tokens 为 full precision，同时用这 32 个 token 计算 weighted k-means 的权重。论文在 m 与 K 上做超参扫描，m=32 平均准确率较优，K=512 准确率饱和且适配 intra-row indirection。性能指标包括 total execution time、per-decoding-step latency、attention/matmul time、memory reduction、accuracy、energy、area。Ablation 显示在 K=128 aggressive compression 下，standard PQ 平均 44.29，去 weighting 为 43.25，去 pre-sort 为 48.76，完整 AQPIM 为 50.00，说明 pre-sorting 与 weighting 对精度保留有实质作用。面积方面，intra-row indirection 额外逻辑为每 HBM 0.0565 mm2，仅为 HBM3 AttAcc! BankPE area 的 0.43%；论文用 FP16 做架构仿真以便与 AttAcc! 对比，同时说明 bfloat16 MAC 延迟相同且面积/能效可能更优。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：论文的核心新执行路径是 AQPIM GPU-PIM 协同的 PQ-based attention pipeline，而不是传统意义上的 GPU kernel。它由 prefilling-time in-memory codebook generation、decoding-time compressed attention、page-aware windowed clustering、intra-row indirection 组成。关键 kernel-like 操作包括 DC(distance calculation)、CA(cluster assignment)、CC(centroid calculation)、ATNK(query-key codebook partial product)、SFM(softmax)、ATNV(attention-value computation)。
        - 新 pipeline/kernel 的执行流例子：以一个长上下文请求的某一层为例，prefilling 时 GPU 先生成该层 Q/K/V，并继续执行 attention、projection、FFN；同时 KV 被 offload 到 HBM-PIM。PIM 中各 BankPE 对本 bank 的 KV subvectors 计算到 centroids 的距离 DC，把距离结果传到 BufferPE；BufferPE 做 CA，选出每个 subvector/token 的 centroid assignment；BankPE 根据 assignment 做 weighted centroid numerator 累加，BufferPE 计算 denominator reciprocal，BankPE 用乘法完成 centroid update。这个过程迭代 4 次生成 codebook，并按 layer-wise 方式压缩 KV，降低 peak memory。
        - decoding 中一条 token 如何流过 AQPIM：GPU 为新 token 生成 q/k/v，把 qkv 发送给 PIM；PIM 将新的 k/v 量化为对应 PQ index 并追加。对 key attention，query 被切成 m=32 个 subvectors，每个 BankPE 用 query subvector 乘对应 key codebook，得到 inner product matrix；key indices 指向各 token/subvector 对应 centroid，PIM 通过 intra-row indirection 在 row buffer 内取出 partial products，并沿 subvector 维度求和，得到近似 qK^T。BufferPE 对 qK^T 做 softmax 得到 attention scores。对 value 路径，PIM 依据 value codebook/value indices 形成 value 贡献，BankPE 执行 ATNV 得到 attention output，最后把 output 传回 GPU，GPU 继续 projection 与 FFN。
        - intra-row indirection 的具体机制：naive PQ attention 的 lookup 会随机访问 inner product table，导致频繁 row activation。AQPIM 先用 page-aware windowed clustering 保证一个 window 内相关 centroid partial products 放在同一个 DRAM row/page 中；运行时激活包含目标 inner product values 的 row，把 lookup indices 放在 GRF 中并重定向给 column decoder，由 column decoder 直接输出对应列值。只要 index 覆盖的地址范围完全位于 row buffer 内，就只需一次 row activation。论文对比 BufferPE gather，指出 BankPE intra-row indirection 能显著减少 off-bank data transfer，尤其 value 路径避免了不必要的 BankPE-BufferPE 往返。
        - pipeline 的边界和限制：AQPIM 没有改变 Transformer 层的整体语义，也没有加速 projection/FFN；它主要优化 KV cache 容量、attention matmul/lookup 和 GPU-CPU/PIM 通信。其精度依赖 PQ 参数、weighted clustering、pre-sorting、保留 sink/recent tokens 的策略；其性能依赖 HBM-PIM row buffer 大小、bank/head/subvector 映射、PIM command 支持和 simulator 假设。论文明确说部分 PIM commands 尚未在 HBM-PIM 中实现，而是通过标准 HBM command path 建模，因此当前结果属于架构仿真与算法评估，不是实物 HBM-PIM 芯片端到端实测。
