    1、论文工作：
        - 论文要解决的核心问题：视频 Vision-Language Models 在推理时会产生大量视觉 tokens，LLM 主体又占据绝大多数参数和计算，导致计算、激活存储和 DRAM 访问压力都很高。以 LLaVA-OneVision 在 VideoMME 上为例，每个样本平均有 6272 个 visual tokens，而文本 tokens 只有 109 个；论文指出 LLM 部分在 LLaVA-OneVision-72B 中占 99.35% 参数和 98.98% operations。因此，若不能在进入后续注意力/FC 层之前减少无效视觉信息，VLM 在边缘或实时部署场景中会受限于算力和内存带宽。
        - 论文的主要贡献：提出 Focus，一个面向 VLM 推理的 streaming concentration architecture。它把冗余消除分成 semantic、block、vector 三个层级：SEC 使用跨模态 attention 做 prompt-aware token pruning；SIC 在 GEMM tile 内用 2x2x2 时空 block 和 vector-wise cosine matching 做细粒度相似性消除；硬件上把 Focus Unit 放在 systolic-array accelerator 的 memory interface 附近，使压缩在 on-chip、streaming、tile-local 路径上完成。论文还提供 PyTorch 算法实现、cycle-accurate simulator、SystemVerilog/Verilog RTL 和复现实验脚本。
        - 论文所处背景：VLM/Video VLM 输入具有强视觉冗余，但冗余的重要性依赖文本问题。例如同一视频中，问题问 dog 时 attention 集中在狗，问题问 flower color 时 attention 转向花的位置。传统 token pruning / token merging 多数只用静态 saliency、magnitude、相似度或视觉 token 间关系，缺乏对 cross-modal semantic intent 的建模；同时 GPU 上 irregular sparsity 难以高效利用，已有 ViT accelerator 又没有专门面向 VLM 的跨模态冗余。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：软件算法 baseline 如 FrameFusion、Prumerge 等能减少 token，但通常有额外 importance estimation / merging 开销，并产生不规则稀疏模式，GPU Tensor Core 很难充分利用。硬件 baseline AdapTiV 主要做 intra-frame token-level similarity，CMC 借助 video codec 做 inter-frame redundancy search，它们都更偏 ViT/视觉冗余，忽略文本 prompt 对视觉 token 重要性的改变。更关键的是，CMC 这类 global token-wise 方法常在完整 token 输出写回 DRAM 后再压缩，带来高带宽和差 locality；论文给出的对比中，CMC 虽有 46% sparsity，但仍有 79% dense DRAM traffic，而 Focus 约 81% sparsity 且只需 21% bandwidth。
        - 论文的设计方法：Focus 使用 Multilevel Concentration。第一层 Semantic Concentrator (SEC) 在 attention layer 中提取 text-to-image cross-modal attention，计算每个 image token 从文本 token 和各 head 中收到的最大 attention score，用 streaming top-k sorter 保留语义相关 tokens，并产生 localized offset encoding。第二层 Similarity Concentrator (SIC) 在 FC / O projection / PV GEMM 的输出 tile 上做 vector-level redundancy removal：以 m=1024、n=32 的 GEMM tile 输出为单位，将 token embedding 分成 32 维 vectors，按 2x2x2 spatial-temporal block 做 localized cosine similarity matching，只存唯一 vector 和 similarity map。第三层通过 scatter/gather 在后续 GEMM 中对 compressed vectors 执行计算，再根据 similarity map 恢复原始 token 位置并累加，保证功能正确。
        - 方法如何对冲 Baseline 缺陷：SEC 直接利用已有 attention pipeline 的 cross-modal 信息，不再依赖与 prompt 无关的静态 token 指标；top-k 过程与 image attention 主计算重叠，避免成为 critical path。SIC 不等整层或全序列 token 就绪，而是在每个 GEMM tile 产生后立即 on-chip 压缩，避免 global token-wise 方法的 DRAM 往返。convolution-style layouter 用 FHW 位置和 bank/offset 映射让 2x2x2 block 的 8 个 vectors 落在不同 SRAM bank 中，支持无复制、无 bank conflict 的并行读取。这样，算法稀疏性被转化为硬件友好的 structured/tile-local 稀疏，而不是留给 GPU 或外部 codec 处理。
        - 关键 trade-off：Focus 接受了额外硬件单元、metadata 和设计约束，换取更低计算和带宽。SEC 需要 top-k 排序、importance buffer 和 offset encoding；SIC 需要 similarity matcher、convolution-style layouter、similarity map、scatter accumulator。tile size、vector size、block size 也形成 trade-off：更小 tile 降低 buffer 但削弱跨 tile 匹配并增加 latency；更小 vector 带来更细粒度冗余但增加 K 维迭代和 accumulator 工作；更大 block 可发现更多相似性但增加匹配复杂度。论文选择 m=1024、vector length=32、block size=2x2x2、similarity threshold=0.9 作为综合点。

    3、论文实现：
        - Baseline 如何实现：算法侧比较 FrameFusion、AdapTiV pruning 和 CMC pruning，并将 AdapTiV/CMC 的 token pruning 思路复现并调参以适配 VLM；FrameFusion 使用官方开源实现。硬件侧比较 vanilla systolic array、AdapTiV 和 CMC，论文用相同技术节点、频率、PE 数量/位宽和 DRAM bandwidth 做公平评估，并将 baseline accelerator 的核心逻辑同样用 SystemVerilog 实现后综合评估面积和能耗。GPU 对照使用 NVIDIA A100 上的 FP16 算法运行，以及 Jetson Orin Nano GPU with/without FrameFusion 的性能对比。
        - 新设计如何实现：Focus 算法用 PyTorch 实现，用 HuggingFace Transformers 和 lmms-eval 生成 accuracy/sparsity 结果及 layer-wise sparse traces。架构性能用基于 SCALEsim-v2 的 cycle-accurate simulation framework 建模 cycles 和 memory access。硬件用 SystemVerilog 实现 Focus architecture，on-chip SRAM 由 TSMC N28HPC+ Memory Compiler 生成，RTL 用 Synopsys Design Compiler 在 28nm 下综合；目标时钟周期 1.32 ns，论文还报告在 500 MHz place-and-route 目标下有 34% timing margin。DRAM 能耗使用 DRAMsim3 建模。
        - 实验 / 实现平台：评估模型包括 LLaVA-Video-7B、LLaVA-OneVision-7B、MiniCPM-V-2.6，视频数据集包括 VideoMME、MVBench、MLVU；扩展到 image VLM 时还使用 LLaVA-OneVision、Qwen2.5-VL 以及 VQAv2、MME、MMBench。Focus 默认硬件配置为 32x32 PE array、FP16 multiply / FP32 accumulate、weight stationary；on-chip buffer 包括 128KB input、78KB weight、512KB output、16KB layouter buffer，总计 734KB；off-chip memory 为 DDR4 4Gb x16、2133R、4 channels、64GB/s。Artifact 要求 Ubuntu 22.04、Python 3.11、PyTorch 2.6.0、CUDA、Transformers 4.48.2/4.49.0、FlashAttention 2.7.4.post1、A100 80GB HBM 和约 128GB 存储。
        - 关键实验设置与指标：主要指标包括 accuracy、computation sparsity、latency/cycles、energy、area、power、DRAM traffic、activation size、ablation speedup、design-space exploration。Focus 平均 accuracy degradation 为 1.20%，平均 sparsity 为 80.19%，相比 AdapTiV 和 CMC 分别高 37.37% 和 31.98%，比 FrameFusion 高 10.19%。性能上 Focus 对 dense systolic array 平均 4.47x speedup，对 AdapTiV 2.60x，对 CMC 2.35x；相对 GPU 为 7.90x，相对 GPU+FrameFusion 为 2.37x。能效上相对 SA、GPU、AdapTiV、CMC、GPU+FrameFusion 分别提升 4.67x、17.09x、2.98x、3.29x、5.13x。面积/功耗方面 Focus 为 3.21 mm2、736 mW，相比 systolic array 只增加 2.7% area 和 0.9% power；SEC/SIC 分别占总面积 1.9% 和 0.8%。内存侧相对 dense SA 压缩 input matrix 5.6x、减少 memory traffic 4.9x；相对 CMC/AdapTiV 分别有 3.7x/2.2x DRAM traffic reduction。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：论文没有提出传统意义上的单个 CUDA kernel，而是提出了一个硬件数据流 pipeline：Streaming Multilevel Concentration pipeline。它由 SEC token-level semantic pruning、SIC vector-level Similarity Gather、Similarity Scatter、convolution-style layout 和 similarity map reconstruction 组成，嵌入 systolic-array accelerator 的 GEMM/attention/FC 执行流中。其核心不是把稀疏性留到软件运行时处理，而是在 attention 和 GEMM tile 产生过程中把无用或相似视觉信息转化为 compact vectors + metadata。
        - 新 pipeline/kernel 的执行流例子：一次视频 VLM 请求先把视频帧切成 visual tokens，并与文本 prompt 一起进入 LLM attention。到 attention layer 时，SEC 从 SoftMax(QK^T) 中取 text-to-image attention block，对每个 image token 计算跨文本 token/heads 的最大重要性分数；a-way streaming bubble sorter 选择 top-k visual tokens，未选中 tokens 在后续 P(i)V 和下游层中不再加载或计算，同时 offset encoder 为保留 token 记录相对位置。进入 FC / projection GEMM 后，systolic array 以 m=1024、n=32 的 tile 输出 32 维 vectors；convolution-style layouter 根据原始 frame/height/width 坐标把 vectors 组织进 2x2x2 时空 block，并用 bank=f mod 2 x 4 + r mod 2 x 2 + c mod 2 这类映射避免 SRAM bank conflict。SIC 选 block 中高 index vector 作为 key，与邻近 7 个 vectors 做 cosine similarity；若匹配，则只记录代表 vector 的 index，若不匹配则写入 compact output buffer。每个 tile 最终只把 deduplicated vectors 和 1xm similarity map 写回 DRAM。下一层 GEMM 对 compact vectors 执行计算，Similarity Scatter 根据上一层 similarity map 把 partial sums 复制/分发回原始 token indices 并在 output-stationary buffer 中累加；tile 完成后再调用 Similarity Gather 做下一轮压缩。这个 gather-scatter 循环让 Focus 在保留模型语义输出的同时减少 PE 输入量、activation 写回量和 DRAM traffic。
