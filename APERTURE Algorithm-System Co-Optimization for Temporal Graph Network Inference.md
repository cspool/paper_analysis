    1、论文工作：
        - 论文要解决的核心问题：
          APERTURE 解决的是 Temporal Graph Network（TGN）推理系统中的逐步执行范式限制。传统 TGN 推理把输入的时间戳交互流切成一系列 temporal graphs，并按时间顺序处理每个 temporal graph：先执行 memory update，更新节点状态和 raw message store，再立刻执行 GNN aggregation 生成当前步的预测结果。这个范式保证了时间因果顺序，但也把 memory update 和 aggregation 强绑定起来，使后一个 temporal graph 的 memory update 必须等待前一个 temporal graph 的 aggregation 完成。论文指出，真正需要严格时间顺序的是 memory update，因为节点状态的演化依赖过去；GNN aggregation 只是读取已经更新好的状态做预测，并不会反向修改 node memory，因此不必成为后续 memory update 的前置依赖。
        - 瓶颈来源：
          瓶颈主要来自执行依赖、GPU 内存访问和粒度选择三方面。第一，step-wise paradigm 限制了跨 temporal graph 的并行机会，导致 aggregation 只能小粒度、频繁 kernel launch。第二，memory update 本身高度 memory-bound，论文的 profiling 显示 load/store 占据 memory update 时间的大头，真正的 update logic 和 time encoding 只占较小比例；当多个 temporal states 被保留下来后，node memory handoff 变成显式的跨图依赖，重复 global memory read/write 更明显。第三，aggregation granularity Gamma 越大，aggregation 的复用和 kernel 合并收益越高，但 temporal states 生命周期也越长，峰值 GPU memory 增大，wiki-talk 这类数据集在较小 Gamma/Gamma_max 下就可能 OOM。
        - 论文的主要贡献：
          论文提出 APERTURE，一个面向 TGN inference 的 algorithm-system co-optimization framework。它的核心贡献包括：1）从语义上证明和利用 memory update 与 GNN aggregation 的可解耦性，在不破坏预测准确率的前提下让多个 temporal states 延后统一 aggregation；2）提出 CG Transformation Engine，用 global feature map、remap-dedup-unmap 和 block-diagonal sparse aggregation 消除 allocator/concatenation 开销、重复特征加载和多次 kernel launch；3）提出 Memory Update Manager，把 node memory / raw message store 隐含的依赖恢复成 state-based DAG，并通过 per-state timedelta materialization 和 cross-level state recomposition 降低 global memory traffic；4）提出 Aggregation Granularity Calculator，用解析模型估计 runtime 与 peak memory，在给定 GPU memory budget 下选择最大安全 granularity。实验中 APERTURE 相比 state-of-the-art baseline 最高达到 59.3x end-to-end speedup，并保持 AP/AUC 与 baseline 一致。
        - 论文所处背景：
          论文面向 dynamic graph 上的 TGN link prediction 推理场景，例如推荐、金融欺诈检测、交通系统等持续演化图。已有系统如 ETC、TGOpt、TGLite 已优化 temporal graph sampling、deduplication、memoization、precomputation 和轻量算子抽象，但仍在 step-wise TGN inference 范式内工作。论文的出发点不是替换 TGN、TGAT、JODIE 等模型结构，而是重写推理执行图：保留 memory update 的时间顺序，同时把只读 aggregation 从逐步路径中拿出来，获得跨 temporal graph 的全局调度和内存优化空间。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：
          TGLite 及其前身 TGOpt 主要减少 temporal GNN 中的冗余 sampling、重复 embedding 计算、time encoding 和数据搬运，但仍把每个 temporal graph 作为独立执行单元。这样做有三类不足。第一，aggregation 以小 temporal graph 为单位执行，kernel launch 次数多，特征加载重复，GPU 利用率不高。第二，memory update 依赖 node memory 和 raw message store 的隐式 handoff，传统逐步处理时这些 handoff 被每步边界隐藏；一旦试图跨步优化，就会暴露大量重复 global memory 读写。第三，baseline 没有全局状态视图来决定一次应该合并多少 memory updates。如果直接做 naive computation graph transformation，即连续执行 Gamma 个 memory updates、物化中间 states、再统一 aggregation，虽然 aggregation 时间下降，但 repeated allocation/deallocation 和 large-state concatenation 会随 Gamma 增长，最终可能抵消收益甚至比不变换更慢。
        - 论文的设计方法：
          APERTURE 的设计可以概括为“语义解耦 + 全局状态布局 + 依赖感知更新 + 内存约束粒度选择”。语义层面，它提出 sufficient conditions：aggregation/read path 不修改状态、不反馈到 update；time encoding 只依赖 timestamps；read aggregation 对 neighbor multiset 满足 permutation invariance。在这些条件下，所有 memory updates 可以继续按时间顺序执行，aggregation 可以延后、重排或并行执行，只要读取相同 snapshot，输出与 step-wise execution 一致。系统层面，APERTURE 先用 global feature map 管理不同 level 的 deduplicated states，然后用 block-diagonal sparse matrix 把多个 temporal graphs 的 aggregation 合成一次 zero-redundancy aggregation。对 memory update，它构建 state-based DAG，显式记录 state 的 parent states，并提前物化每个 state 所需的 timedelta；相邻 level 若存在可组合 producer-consumer 关系，则在一个 fused kernel 中前向传递中间结果。最后，它用 peak memory model 估计给定 Gamma 下 GlobalState、TimeDelta、Aggregation workspace 和固定开销，选择不超过 M_max 的最大 granularity。
        - 方法如何对冲 Baseline 缺陷：
          对逐步范式的并行性不足，APERTURE 删除的是“不必要的 aggregation 前置依赖”，而不是删除 memory update 的因果顺序，因此能保留 TGN 语义并扩大执行窗口。对 aggregation 冗余，它不再对每个 temporal graph 单独 sparse aggregation，而是构造 block-diagonal adjacency，把多个图的 adjacency 沿对角线拼接，用一次矩阵乘法读取 global feature map，避免重复 feature loading 和多次 kernel launch。对 memory-bound update，它用 state-based DAG 显式化 NM/RMS 依赖，让相邻 level 的可组合状态在寄存器或 shared memory 中传递，只把最终状态写回 global memory，从而减少中间 state 的 spill。对 granularity 风险，它不靠昂贵 autotuning，而是在 preprocessing 后用 deduplicated-node upper bound、fanout、feature bytes、state/time dimensions 和 aggregation workspace 模型做 conservative peak memory estimation，优先保证 memory safety。
        - 关键 trade-off：
          APERTURE 接受了更复杂的 preprocessing、state metadata、global feature map、DAG construction 和解析内存模型，以换取跨步 aggregation 复用和 memory update traffic 降低。Gamma 越大通常运行时间越低，因为 synchronization point、aggregation 次数和 kernel launch 减少，但中间 temporal states 生命周期更长，峰值显存更高；因此系统需要保守估计 memory peak，可能牺牲一部分可用显存空间来避免 OOM。DAG construction 和 time materialization 也带来额外开销，论文 Figure 16 显示 DAG construction 是主要 overhead，但在所有 workload 中仍低于 end-to-end time 的 10%。此外，论文当前主要评估单 GPU；未来工作才计划扩展到 multi-GPU，说明 partitioned feature maps、DAG-guided updates 和 boundary exchange 尚未在本文中作为已完成系统验证。

    3、论文实现：
        - Baseline 如何实现：
          论文把 APERTURE 与 state-of-the-art TGLite 对比，并区分 TGLite-CPU 和 TGLite-GPU 两个实现。TGLite-CPU 中，node/edge features、node memory、raw message store 等 state tensors 放在 host memory，需要按需拉取到 GPU；TGLite-GPU 中，这些张量常驻 GPU，以避免 host-to-device transfer，但在 gdelt 等大图上可能因为多个 tensor 常驻而 OOM。所有方法使用相同配置以保证公平。论文还在动机部分实现了 Naive-CG：连续执行 Gamma 个 memory updates，物化中间 node states，再执行一次 GNN aggregation，用来说明单纯 computation graph transformation 会被 memory management 和 state concatenation 开销拖垮。
        - 新设计如何实现：
          APERTURE 由三个主要模块协同执行。CG Transformation Engine 将 sampling 迁移到 GPU，并执行 remap-dedup-unmap preprocessing：remap 通过 prefix-sum 为跨 temporal graph deduplication 分配 offset，dedup 后 unmap 恢复原始 per-graph partition，既保留 temporal graph 边界，也避免 workload imbalance。Global Feature Map 以 state 为单位存储中间数据，每个 state 由 node_id、level_id、uniq_id 描述，地址通过 level 的 prefix sum 和 uniq_id 计算，类似 CSR 风格紧凑布局。Zero Redundancy Aggregation 通过 block-diagonal sparse matrix 将多个 temporal graph 的 adjacency 拼接，使用一个 aggregation kernel 完成多步 read-only aggregation。
        - Memory Update Manager 的实现：
          Memory Update Manager 首先构建 state-based DAG。它初始化两个 T x N 表：Nid-level table 记录某节点在某 level 之前最近一次 NM 更新 level，DNid-LNid table 记录 raw-message update 中 destination node 与 latest neighbor 的配对。随后一个 state 对应一个 thread 执行 parent lookup：对 state (level, node) 找到最近 NM parent，并根据 DNid-LNid 判断是否还有 sibling parent。Timedelta materialization 复用同样的 parent lookup 逻辑，预先得到 RMS time 与 NM time 的差值并执行 time encoding，避免每次 update 重复访问 NM/RMS timestamp。Cross-level state recomposition 则扫描 state DAG，识别 level l+1 中依赖 level l 唯一 producer 的 composable subset；执行时 fused kernel 计算 level l 的 update，并通过 register/shared memory 把结果转发到 level l+1，只把最终状态写回 global memory。不可直接 fuse 的部分则利用 kernel 内较早完成的线程做 global-to-global 数据搬运，减少额外 kernel launch。
        - 实验 / 实现平台：
          论文使用三种代表性 temporal graph models：TGN、TGAT、JODIE。数据集包括 lastfm、reddit、wiki、mooc、wiki-talk 和 gdelt，其中 wiki-talk 和 gdelt 是大规模数据集；例如 wiki-talk 有 1,140,149 nodes 和 7,833,140 edges，gdelt 有 16,682 nodes 和 191,290,882 edges。硬件主平台是 Intel Xeon Gold 6336Y CPU + NVIDIA A100 40GB GPU；为了展示硬件可移植性和 memory safety，还在 Intel Xeon Gold 6230R CPU + NVIDIA RTX 4090 24GB GPU 上报告 end-to-end 结果。数据划分遵循 prior work，train/val/test 为 70/15/15；小数据集 temporal graphs group size 为 200，大数据集为 2000；每个实验重复 5 次取平均。
        - 关键实验设置与指标：
          主要性能指标是相对 TGLite-CPU / TGLite-GPU 的 end-to-end speedup。A100 上，APERTURE 平均比 TGLite-CPU 快 29.19x，比 TGLite-GPU 快 21.15x；RTX 4090 上，平均比 TGLite-CPU 快 34.53x，比 TGLite-GPU 快 27.85x；最高 speedup 为 59.3x。准确性指标使用 AP 和 AUC，论文 Table 3 显示 APERTURE 与 TGLite 在 TGN、TGAT、JODIE 和所有数据集上 AP/AUC 相同或仅四舍五入差异，说明解耦没有改变预测质量。Breakdown 分析把 runtime 分为 GA、TE、MU、SD、LD：APERTURE 的主要效果是让原本占主导的 GNN Aggregation 和 Memory Update 降到个位数百分比。Granularity 分析显示 memory estimator 平均相对误差为 14.14%，偏保守但能避免 OOM；Gamma 增大时 end-to-end time 一般下降。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：
          APERTURE 引入的是一条 decoupled TGN inference pipeline，而不是单个孤立算子。它的核心执行路径包括：1）按 temporal order 执行多个 memory updates，但把 aggregation 延后；2）用 CG Transformation Engine 将多个 temporal graphs 的 states 放进 global feature map；3）用 zero-redundancy aggregation kernel 对 block-diagonal sparse adjacency 和 global feature map 执行一次统一 aggregation；4）用 Memory Update Manager 的 state-based DAG 与 cross-level state recomposition kernel 降低 memory update 的 global memory traffic；5）用 Aggregation Granularity Calculator 在显存预算下选择最大的安全 Gamma。最接近“新 kernel”的部分有两个：block-diagonal zero-redundancy aggregation kernel，以及跨 level state recomposition 的 fused update kernel。
        - decoupled TGN inference pipeline 的执行流程：
          一个 temporal interaction stream 到来后，系统仍按时间顺序构造 temporal graphs，并执行 temporal neighbor sampling。不同于 baseline 逐图执行“memory update -> aggregation -> 下一图 memory update”，APERTURE 先对一段长度为 Gamma 的 temporal graphs 做 preprocessing，得到每个 level 中 deduplicated states 的数量、状态位置和依赖关系。随后 memory update 仍按 temporal order 推进，保证 node memory 的因果演化正确；但每一步更新后的 temporal state 不立刻触发独立 aggregation，而是被写入 global feature map 中由 (level_id, uniq_id) 定位的槽位。等 Gamma 个 memory updates 准备好后，系统将各 temporal graph 的 sparse adjacency 拼成 block-diagonal matrix，用一次 aggregation 操作对所有只读 states 生成预测 embedding。
        - 一个 state 如何流过系统：
          假设 level 2 上节点 v=1 需要更新。APERTURE 的 DAG construction thread 会先在 Nid-level table 的 v=1 列向前查找，找到该节点最近一次 NM update 所在 parent level；再在 DNid-LNid table 中判断这个 destination node 是否还携带 latest neighbor parent。如果存在 raw message 配对，就把 sibling parent 也记录进 parent_states。Timedelta materialization 复用这个 parent key，从 RMS-time 和 NM-time 表里回溯出对应 timestamp，计算 delta_t 并做 time encoding。执行 update 时，如果 state DAG 表明 level 2 的这个 state 是 level 3 某 state 的唯一 producer，cross-level recomposition kernel 可以在完成 level 2 update 后，不把中间结果写回 global memory，而是通过寄存器或 shared memory 直接传给 level 3 update；最终只把 level 3 的结果写回 global feature map。
        - zero-redundancy aggregation kernel 的执行例子：
          假设 Gamma=3，系统积累了三个 temporal graphs G1、G2、G3。Baseline 会对 G1、G2、G3 分别执行 aggregation，每次读取相应节点特征、edge index、time encoding，并启动独立 sparse aggregation kernel。APERTURE 则把三个图的 adjacency 沿对角线拼成一个大的 block-diagonal sparse matrix，三个图的 states 已经按 level 连续存储在 global feature map 中。一个 kernel launch 即可完成所有图的 aggregation：block diagonal 结构保证不同 temporal graph 之间不会错误消息传递，global feature map 保证重复 state 已经 deduplicated 并有唯一存储位置。这样既保持每个 temporal graph 的语义边界，又减少重复 loads、重复 computation 和 kernel launch overhead。
        - cross-level state recomposition kernel 的执行例子：
          假设 state s_l 在 level l 更新后立即被 level l+1 的某个 state 作为唯一 parent 使用。传统执行会把 s_l 写入 global memory，下一步再从 global memory 读出，执行 level l+1 update，然后再写回。APERTURE 的 fused kernel 在同一个 kernel execution 内完成两级更新：线程先计算 level l 的 update，用已经物化好的 timedelta 和 parent metadata 得到中间 state；如果 DAG 判断它属于 composable subset，就直接通过 register/shared memory 转发给 level l+1 的 consumer；level l+1 update 完成后，只把最终 state 写入 global memory。这个路径减少了一次中间写和一次后续读，也减少单独 kernel launch。
        - 与传统 pipeline/kernel 的区别：
          传统 TGN inference pipeline 的调度单位是 temporal graph，依赖边界是完整的“update + aggregate”步骤；APERTURE 的调度单位变成 temporal state 和 aggregation granularity。传统系统即使做 deduplication、memoization 或 GPU 常驻，也仍缺少跨 temporal graph 的全局 feature map 和 state DAG，因此很难在保证因果正确性的同时合并 read-only aggregation、消除中间 update traffic。APERTURE 的关键不是把时间顺序完全并行化，而是精确地区分“必须顺序”的 update path 和“可延后并行”的 read path；在这个区分之上，它用 global feature map、state-based DAG 和 memory-bounded granularity selection 把算法语义变成可执行的系统优化。
        - 局限与可迁移性：
          APERTURE 的性能收益依赖 temporal graph 中的重复节点、aggregation 冗余和 memory-bound update 特征；在更稀疏、重复度更低或 aggregation 本身不占主导的 workload 上，收益可能主要来自 kernel consolidation，而非大幅冗余删除。Granularity estimator 是保守的，它用安全 margin 避免 OOM，但可能低估可用 Gamma。论文当前实验集中在单 GPU，multi-GPU 只作为 future work；如果扩展到多 GPU，需要解决 global feature map partition、state DAG boundary exchange 和跨 GPU temporal dependency 的一致性问题。
