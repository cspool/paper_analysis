## LRM-GPU: Alleviating Synchronization Overhead for Multi-Chiplet GPU Architecture

- 属于硬件架构的实现是什么？实验比较什么？
  提出LRM-GPU硬件架构，为multi-chiplet GPU提供高效同步支持。实现包含两大机制：(1) Lazy Release Consistency on Multi-Chiplet GPU——在LLC中实现synchronization variable directory（sync-val directory），跟踪每个同步变量的owner chiplet，仅在owner跨chiplet迁移时才对额外cache层（L1.5）触发invalidate/flush coherence action，利用chiplet内同步locality减少冗余cache操作；(2) In-Network Synchronization Atomic Merge——在每个chiplet网络中嵌入Synchronization Atomic Merge Unit (AMU)，由merge table (CAM+SRAM dual-port)、instruction decoder、ALU、multicast unit组成，检测并合并跨chiplet的同地址atomic请求（如多个atomicAdd合并、多个atomicCAS合并为一次传输），减少inter-chiplet同步atomic traffic。实验比较：vs MCM-GPU baseline（传统GPU同步方案）、hLRC（同步变量多级cache追踪）、HMG（面向multi-GPU的cache coherence协议扩展）、AMU-only。LRM-GPU相对MCM-GPU平均加速1.33×（其中AMU贡献1.16×），相对HMG平均加速1.22×，减少L1.5 cache invalidation 30%，减少inter-chiplet traffic 28%（相对HMG减少52%），减少能耗18%（相对HMG减少32%）。对无全局同步的workload（VGG16、GPT-2等）性能影响仅~2%。

- 模拟器名，模拟器链接（web search），或论文修改的模拟器。
  扩展GPGPU-Sim（开源GPU模拟器，https://github.com/gpgpu-sim/gpgpu-sim_distribution）模拟multi-chiplet GPU系统。基于BookSim 2.0（开源NoC模拟器，https://github.com/booksim/booksim2）构建chiplet interconnection platform，支持composable network construction、heterogeneous router/crossbar microarchitecture、network integration、modular routing和chiplet interface incorporation。该platform集成到GPGPU-Sim中，实现whole multi-chiplet system的cycle-accurate uniform evaluation。AMU merge table部分使用Cadence Virtuoso定制电路（TSMC 40nm），其他AMU组件用Verilog RTL + Synopsys Design Compiler综合（TSMC 40nm）。能耗评估使用AccelWattch（GPGPU-Sim power model）配合inter-chiplet传输能量数据（0.54 pJ/bit）。论文未提供模拟器修改和RTL的开源链接。

- 模拟器模拟什么的性能，修改了什么。
  模拟器模拟4-chiplet GPU系统（256 SMs, 64 SMs/chiplet），每SM 128KB L1 write-through cache，每chiplet 2MB L1.5 write-through cache（仅缓存remote data），全局8MB LLC write-back cache。Inter-chiplet network为concentrated hierarchical crossbar拓扑，带宽768GB/s、32 cycles/hop。DRAM 64 channels、3TB/s。采用first-touch page allocation（4KB pages）和distributed CTA scheduling。论文修改GPGPU-Sim：(1) 实现LRM-GPU lazy release consistency coherence action逻辑——sync-val directory共64 entries（16 entries/chiplet），记录同步变量owner chiplet，决定acquire/release时的L1.5 invalidate/flush策略（四种场景：invalid/local/remote/evicted）；(2) 在chiplet网络中嵌入AMU——16 channels、merge table 2K entries/16 banks、每entry SM list最多8 SM IDs、data 32B，实现atomic请求merge/multicast响应逻辑；(3) 基于BookSim 2.0构建composable multi-chiplet network。模拟器输出：execution time（计算speedup）、L1.5 cache invalidation count、inter-chiplet traffic、energy consumption、chiplet scale/latency sensitivity。

- 开源情况。模拟器如何使用？作用是什么？基于开源文档和论文，使用例子解释。
  开源：LRM-GPU修改和AMU RTL未开源。GPGPU-Sim开源、BookSim 2.0开源。模拟器使用流程：
  1. 系统配置：设置4-chiplet系统参数（SM数、cache size/policy、NoC拓扑、inter-chiplet bandwidth/latency、DRAM channels/bandwidth、page allocation策略、CTA scheduling策略）→ 配置LRM-GPU参数（sync-val directory entries/chiplet、AMU channel数/merge table entries/banks/SM list size）
  2. Workload编译：CUDA 11.1 + O3编译microbenchmarks（atomicTreeBarr/lfTreeBarr/spinMutex/sleepMutex/faMutex/spinSem）+ 全局同步benchmarks（reduce/scan/histogram/pagerank/barnes-hut/hash-table/MST）+ 无全局同步benchmarks（b+tree/backprop/bfs/dwt2d/nn/lavaMD/VGG16/GPT-2/hotspot）
  3. 模拟执行：GPGPU-Sim执行workload kernels → LRM-GPU sync-val directory在LLC拦截acquire/release同步操作 → 判断同步变量owner状态（invalid/local chiplet/remote chiplet/evicted）→ 仅当owner跨chiplet迁移时触发L1.5 flush/invalidate → AMU在网络中拦截跨chiplet atomic请求 → merge table CAM查询同地址entry → ALU合并 → timer到期/SM list满后发送合并请求 → 响应通过multicast unit广播到所有参与SM
  4. 输出指标：normalized speedup（以MCM-GPU为baseline）、L1.5 invalidation数、inter-chiplet traffic（含memory access和coherence message）、system energy（AccelWattch + inter-chiplet 0.54 pJ/bit）
  5. 灵敏度分析：sweep chiplet数（4→6→8）和inter-chiplet latency（8→48 cycles）→ 观察LRM-GPU加速比随chiplet数增加而递减（locality下降）
  6. AMU功耗面积验证：Cadence Virtuoso定制merge table → TSMC 40nm netlist → Cadence Spectre仿真（TT corner, 25°C, 1GHz, 1.1V）→ 测量功耗185.51mW/面积1.52mm²；其余AMU逻辑用Synopsys Design Compiler + 40nm标准单元库综合 → 功耗115.93mW/面积0.32mm²；总AMU功耗301.44mW/面积1.84mm²（参考V100 300W/815mm², AMU仅占~0.1%功耗/~0.2%面积）
