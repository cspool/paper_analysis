## AQPIM: Breaking the PIM Capacity Wall for LLMs with In-Memory Activation Quantization

- 属于硬件架构的实现是什么？实验比较什么？
  提出AQPIM硬件架构，基于HBM3-PIM（图7）修改AttAcc!设计，核心硬件创新：(1) BankPE+BufferPE双PE架构：BankPE置于DRAM die bank旁（ADD/MUL/SUM units，面积极度受限，0.43% area overhead from intra-row indirection），BufferPE置于buffer die（MIN/DIV/EXP units，用于data-intensive CA和SFM操作）——匹配AttAcc!已有的accumulators和softmax units；(2) Intra-row indirection硬件：GRF (General-purpose Register File)存储lookup indices→MUX重定向到column decoder→column decoder输出对应inner product values→stream到buffer die或GRF，完全利用现有1KB row buffer和column decoder，仅添加MUX和GRF datapath routing（area 0.0565mm² per HBM, 0.43% of BankPE）；(3) 新增PIM Commands：PIM_SET_CONFIG (broadcast PQ params), PIM_MAC_AB (all-bank MAC), PIM_SFM (softmax in BufferPE), PIM_RET (row buffer retrieval), PIM_MV_BA (Bank→Buffer data move), PIM_MV_BF (Buffer→Bank data move), PIM_RD/PIM_WR (I/O), PIM_ACT_AB (all-bank DRAM row activation) —— 通过标准HBM command path发出，未在商用HBM-PIM实现；(4) Data mapping：head-wise (per HBM) + subvector-wise (per bank)，利用低位memory address做bank selection；(5) Memory allocation：固定codebook region + 固定buffer region (prefill阶段) + layer-wise PQ index region at page granularity，prefill buffer region在decoding阶段被overwrite。实验比较：(左) vs Architecture: GPU vs AttAcc! vs AQPIM的total execution time (图11)和per-decoding-step latency (图12)；AQPIM vs AttAcc!在S_len=32768时per-step latency为0.12× vs 0.19× (vs GPU 1.0×)；(右) vs Algorithm: GPU上的PQCache/SKVQ/SnapKV vs AQPIM，AQPIM在S_len=32768时per-step latency仅0.12×；(图13) speedup breakdown: gpu+cpu vs gpu∞ vs gpu+pq vs aqpim，显示消除offloading penalty (0.18×)和PQ compression (0.047×)的贡献；(图14) energy: AQPIM每decoding步energy仅为GPU的0.18× (S_len=4096)到0.07× (S_len=32768)。

- 模拟器名，模拟器链接（web search），或论文修改的模拟器。
  构建customized GPU-PIM simulator for LLM inference，基于AttAcc! simulator ([55] https://github.com/scale-attacc-kr/attacc-sim) 和Ramulator2 ([20] https://github.com/CMU-SAFARI/ramulator2)。论文未单独开源AQPIM simulator。AttAcc! simulator本身是Ramulator的modified version适配LLM inference模拟。Intra-row indirection logic synthesized with same PDK [9] as AttAcc!, scaled with DRAM die density ratio。

- 模拟器模拟什么的性能，修改了什么。
  模拟GPU+PIM异构系统的LLM inference性能：接受system configuration (H100 GPU specs, HBM-PIM banks/PEs), model details (layer count/heads/hidden dim/FFN dim), input configurations (batch_size/input_length/output_length)作为输入，输出execution time和energy consumption。修改内容：(1) in AttAcc! simulator: 新增BankPE/BufferPE dual-PE model, PQ-based computation pipeline (DC→CA→CC迭代), PQ-attention (ATNK→SFM→ATNV), intra-row indirection datapath, new PIM commands, page-aware memory mapping; (2) DRAM traffic和contentions完全由DRAM controllers管理并在simulator中建模。Timing/energy基于AttAcc! synthesis结果，column decoder input latched for pipelining (tCCDL不变)。

- 开源情况。模拟器如何使用？作用是什么？基于开源文档和论文，使用例子解释。
  AQPIM simulator未独立开源。基于开源的AttAcc! simulator和Ramulator2。Simulator作用：评估AQPIM的performance (execution time breakdown by prefill/d_pq/d_attn/d_fc/d_comm/d_etc) 和energy consumption per decoding step，与GPU baseline、AttAcc! PIM architecture、PQCache/SKVQ/SnapKV compression algorithms进行对比。使用例子：配置HBM-PIM parameters (Nsubvecs=32, Ncentroids=512, batch_size=16 for default) → 输入Mistral-7B-Instruct-v0.2 model specs → 运行4种scenario (gpu+cpu/gpu∞/gpu+pq/aqpim) → simulator输出Fig.13的latency breakdown (12.5ms→0.047ms KV cache memory=120GB→0GB) 和Fig.14的energy breakdown。
