## Difflow: A Data-Characteristic-Aware Serving System for Diffusion Models

- 属于编译框架的实现是什么？实验比较什么？
  提出Difflow（原名ChituDiffusion）编译器，将扩散模型pipeline编译为多个specialized execution engines (dEngines)。核心编译实现：(1) Symbolic Data Property Propagation：为每个tensor操作定义符号化属性传播规则(Table 1)，覆盖Unary elementwise (ReLU/Tanh)、Binary elementwise (+/−/×/÷)、Linear (Batch Matmul)、Convolution (Conv2D w/o padding, w/ padding)等算子——输入输出表示为per-dimension property vectors，redundancy用boolean algebra传播（如Binary elementwise输出[a₁∧b₁, a₂∧b₂]）；(2) dGraph Identification：基于符号化传播，将pipeline DFG按consecutive operators with identical output property expressions分组为dGraphs——因为相同properties暗示shared optimization。对denoising loop先unroll initial iterations直到loop inputs stabilize（converges within few steps）；(3) Selective dEngine Generation：对每个dGraph枚举input property expressions的可能值，为每种satisfiable property condition编译一个specialized dEngine——两个剪枝策略：conflicting conditions (unsatisfiable property combinations, 如tensor B和E共享同一property不能假设不同) 和inessential conditions (speedup <5% threshold)；(4) Redundancy Elimination Rules：dimension-level规则在编译时消除冗余计算——冗余维度的tensor被mark并在后续通过broadcasting恢复，保证等价性。对redundant memory access（如attention中冗余K/V），利用线性代数等价变换将tensor压缩concat为无冗余形式(Figure 6)；(5) Ragged Operation Regularization：将ragged data-sharing operations通过transpose+reshape变换为regular operators——如ragged Matmul通过fuse batch dim+ragged dim compact为regular Matmul (Figure 9a)；ragged elementwise和convolution通过transpose+im2col转换(Figure 9bc)；ragged data-independent operations保持embarrassingly parallel执行（round-robin tile mapping）；(6) Invariant Tensor Elimination：lightweight four-state detection algorithm (constant/loop-invariant/loop-variant/unknown)，constants compile-time precomputed，loop-invariants hoisted outside loops，支持multi-value constants；(7) Architecture-Agnostic DFG IR：以Data Flow Graph为universal intermediate representation，解耦optimization logic与specific NN layout，优化规则作用于individual operators level。实验比较throughput对比PyTorch/Inductor/TensorRT/Stable Fast/Katz；ablation study隔离compile-time optimizations (dGraph scheduling = +SCH, dEngine compilation = +COMP, invariant tensor elimination = +IRE)的逐项贡献；Table 3展示dGraph recomposition将SDXL U-Net编译从monolithic 16384 engines/11天降至4 engines/7分钟。

- 开源编译框架是什么。修改了什么。
  基于PyTorch FX graph + Triton[59] + FlashAttention[21,22] + Stable Fast[1] + Diffusers[61]组件实现，复用torch.compile backend用于DiT compile。Difflow编译器在之上新增：(a) 符号化属性传播引擎——symbolic variables初始化自application characteristics，propagation rules按operator category维护per-dimension boolean expression；(b) dGraph recomposition——unroll+stabilize loops→traverse DFG→group consecutive operators with identical output property expression→post-process merge small dGraphs；(c) dEngine selective compilation——enumerate property conditions→prune conflicting+inessential cases→compile each satisfiable condition to specialized dEngine；(d) 冗余消除规则库——dimension-level rules per operator type，线性代数等价变换规则处理memory access redundancy；(e) ragged operation regularization规则——graph transformation rules (transpose/reshape/im2col)将ragged data-sharing operations转为regular form；(f) invariant tensor四态检测算法——从tensor definitions初始化→iterative priority hierarchy propagation along operators。

- 开源情况。编译框架如何使用？作用是什么？基于开源文档和论文，使用例子解释。
  开源：https://github.com/thu-pacman/chitu/tree/Diffusion。编译器使用流程：
  1. Pipeline定义与符号初始化：用户定义diffusion pipeline（如SDXL+Refiner, SD15+ControlNet）→开发者标注application characteristics（哪些输入fixed property、哪些symbolic variable、哪些varying）
  2. Symbolic Propagation：编译器对DFG执行symbolic property propagation→每个operator的输入/输出property表达为symbolic boolean expression vectors（如dimension redundancy用boolean variable表示）
  3. dGraph Identification：按identical output property expression分组consecutive operators→denoising loop先unroll到stable→small dGraphs merged→例如SDXL U-Net分解为3个dGraphs（input-dependent U-Net层/input-independent U-Net层/post-processing）
  4. dEngine Compilation：对每个dGraph枚举property conditions→prune unsatisfiable+inessential cases→compile surviving conditions为dEngines。例如dGraph(d-e)有2个conditions(β=T, β=F)→生成2个dEngines覆盖所有优化场景
  5. Redundancy + Raggedness Optimization：冗余消除——attention中冗余K/V compress K/V沿redundant batch dim + concat Q tensors→standard attention kernel执行；ragged regularization——ragged Matmul通过fuse+reshape→regular Matmul kernel执行
  6. 例如SDXL U-Net with ControlNet：monolithic策略须枚举2^14=16384 property combinations(estimated 11天编译)，Difflow通过符号化分析识别ControlNet inputs共享相同优化条件→仅需7个dEngines(7分钟)。difflow通过dGraph decomposition将编译开销从组合爆炸降至线性。

Difflow编译器的作用：通过将diffusion pipeline分片为dGraphs并选择性编译为多版本dEngines，在有限编译开销下实现对多种输入数据属性组合的覆盖优化。相比monolithic pipeline optimization的组合爆炸，dGraph decomposition+selective dEngine generation+pruning策略将优化空间控制在可行范围。
