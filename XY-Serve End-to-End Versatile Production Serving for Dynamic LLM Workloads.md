论文标题：XY-Serve: End-to-End Versatile Production Serving for Dynamic LLM Workloads

    1、论文工作：
        - 论文要解决的核心问题：生产级 LLM serving 同时面对输入长度、输出长度、prefix reuse 命中长度、speculative decoding 验证长度、P/D/V 阶段混合比例等多层动态性。这些动态性会直接破坏 AI accelerator，尤其是 Ascend NPU 这类 tile-based 架构上的高效 kernel 假设：Linear 的 GEMM 形状变成任意形状，Attention 的矩阵形状和 mask 结构从规则三角形扩展到矩形、梯形和 tree-based speculative mask，Decode 的 KV cache 访问又受 PagedAttention 分块影响而更加离散。
        - 论文的主要贡献：提出 XY-Serve，一个基于 vLLM 改造的 Ascend-native 端到端 LLM serving 系统。它用 token-wise scheduling 把 Prefill、Decode、Verify token 放进统一 chunk，再通过 Task Decomposition 和 Task Reordering 把动态 workload 转换成硬件友好的 tile-level meta-primitives；在此之上实现 `Meta-Attention` 和 `SmoothGEMM` 两类核心 kernel。实验中 Attention kernel 相比 torch-npu PFA/IFA 平均快 21.5%，Linear kernel 相比现有库平均快 14.6%，端到端在公开 vLLM nightly benchmark 上相比 Ascend-vLLM 最高提升 89%，在内部工业 workload 上最高提升 95%。
        - 论文所处背景：LLM inference 通常分为 prefill 和 decode。实际系统为了提升效率会引入 Automatic Prefix Caching / Prefix Reuse、PagedAttention、Chunked Prefill / SplitFuse、Speculative Decoding、P/D disaggregation 等优化。这些优化单独看能减少计算或改善 latency-throughput trade-off，但组合到生产系统后会把 workload 变得更不规则。GPU 的 SIMT 编程模型可以较自然地处理边界条件和分支，而 Ascend NPU 这类 coarse-grained tile-based 架构需要固定宽度 tile 上执行同一操作，遇到不规则 shape 和 mask 通常要 padding、unpadding 或额外 mask 处理，造成额外计算和访存开销。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：第一，torch-npu 的 PromptFlashAttention 对理想 square prefill attention 效率较好，但 prefix reuse 使 Q/KV 长度变成任意矩形后，tiling 会产生大量 irregular units，论文报告 MFU 从理想场景下降到 47% 甚至 30%，prefix reuse 的理论计算节省被 kernel 效率下降抵消。第二，Verify attention 在 speculative decoding 中有复杂 mask，torch-npu 只能退回到带 full mask 的 prefill kernel，缺少 verify-specific 优化，MBU 低于 30%。第三，Decode attention 本身 memory-bound，PagedAttention 的小 block KV cache 访问进一步降低带宽利用率。第四，通用 torch-npu linear / GEMM 要覆盖所有动态 M/N/K 形状，性能不稳定，难以在每个 shape 上达到最优。第五，若按 batch-by-batch 或 selective batching 分别调用 P/D/V attention kernel，需要 split、rearrange、merge 数据，论文测得这些内存操作在混合 P/D/V 场景中可占到超过 50% 的 latency。
        - 论文的设计方法：XY-Serve 先在 serving 层做 token-wise scheduling，从队列中按固定 budget 选取 token chunk，chunk 可以同时包含 prefill、decode、verify token；长 prefill 被拆成多个 chunk，同时保留部分 slot 给 decode/speculative token 以稳定 TBT。随后系统生成 `Token-Table`，记录每个 stage 或 operator 的 start、tokenNum、kvLen、tileSize 等元数据。Task Decomposition 将 Attention 的 P/D/V stage 逻辑分解成 tile blocks，将 Linear 的 QKV、OProj、GateUp、Down 映射到固定 tile size 的 GEMM 子任务。Task Reordering 再生成 `Task-Table`，把 tile 分配给 AI cores：Attention 按 tileSize * kvLen 估计负载并做对称 round-robin 分配，Linear 则基于离线 profile 的固定 shape 策略做任务顺序和 swizzling 类优化。
        - 方法如何对冲 Baseline 缺陷：对 Attention，XY-Serve 不再为 prefill、decode、verify 的每种组合分别写 kernel，而是把它们统一成 GEMM-Softmax-GEMM 的 meta tile，再在 kernel 内根据 metadata 处理 KV cache layout、mask 和 pipeline 选择。这样 P/D/V mixed batch 不必反复拆分合并，prefix reuse 和 chunked prefill 也能直接利用 block table 读取历史 KV。对 Linear，XY-Serve 不尝试为所有任意 GEMM shape 做在线优化，而是观察到模型参数维度 N/K 由 hiddenSize、intermediateSize 和 TP 决定，通常是 tile size 的倍数，真正动态的是 tokenNum 对应的 M 维；因此它用 on-chip virtual padding 加 selective read/write 把任意 M 映射到少量固定 tile shape，再复用离线优化好的高性能 GEMM primitive。
        - 关键 trade-off：XY-Serve 接受了更复杂的 runtime metadata 和调度路径，包括 Token-Table、Task-Table、block-wise/token-wise KV cache 管理、离线 GEMM shape profiling，以及针对 Ascend intrinsic 的 C++ operator 实现。它牺牲了一部分通用性和实现简洁性，换取生产 workload 下对多种动态优化组合的端到端兼容和稳定硬件利用率。论文也显示 greedy task assignment 比对称 round-robin 只带来边际收益，额外 reordering overhead 会抵消改进，因此系统选择较轻量的启发式策略。

    3、论文实现：
        - Baseline 如何实现：Attention baseline 使用 torch-npu 2.1 中的 `PromptFlashAttention` 和 `IncreFlashAttention`，Linear baseline 使用 torch-npu 的通用 linear/GEMM operator。端到端 baseline 是 Ascend-vLLM / vLLM-npu，它已经支持 vLLM 的主要优化，包括 Chunked Prefill、PagedAttention 和 Prefix Reuse，并使用 torch-npu GEMM 与 fused attention operator。GPU generality 实验中 baseline 使用官方 Triton fused attention 和 matrix multiplication kernels。
        - 新设计如何实现：作者基于 vLLM 构建 Ascend-native inference system，用 Ascend intrinsic 实现 `SmoothGEMM`、`Meta-Attention`、normalization、activation、embedding 等核心 operator，并通过 pybind11 暴露 Python API。为减少频繁 Python 调用开销，论文把整个 model-forward 下沉到 C++，将多个优化 operator 集成到一个 C++ function，再由 vLLM 调用。系统还替换了 vLLM 原生 scheduler，加入 token-wise scheduling、workload decomposition 和 task reordering，并重写 speculative decoding 框架，使其能构建 token tree 和生成 Meta-Attention 所需 metadata，支持 Lookahead Decoding 等 tree-based speculative decoding 算法。
        - 实验 / 实现平台：主平台是 Huawei Ascend 910B NPU，架构上包括 AI Core / Cube Unit、AI Vector、Memory Transfer Engine、L2 cache 和 HBM。端到端公开 benchmark 使用 vLLM nightly-benchmarks，模型为 Qwen2-7B，场景包括 ShareGPT、Sonnet Prefill-heavy、Sonnet Decode-heavy，并在 QPS=4/8/16/32 和 dynamic QPS 下测量。内部工业 workload 使用平均输入长度 2169 tokens 的 Llama-like 66B 模型，TP=8，8-concurrent request。GPU 对比使用 Nvidia A800 和官方 vLLM-v0.6.4.post1；GPU generality 实验把 Meta-Attention 和 SmoothGEMM 迁移到 Triton/JIT 平台。
        - 关键实验设置与指标：Attention 实验包括 P/D/V mixed batch、arbitrary prefix reuse、long-sequence chunked prefill、verify attention、decode attention。指标包括单层 attention latency、MFU、MBU 和 relative latency。SmoothGEMM 实验用 Llama2-7B、Llama2-13B TP=2、Qwen2.5-32B TP=4、Qwen2.5-72B TP=8 等模型形状，输入长度从 1 到 budget length 4096，测 QKV/OProj/GateUp/Down 的 MFU。端到端指标包括 achieved QPS、TTFT、TBT。论文报告：P/D/V mixed attention 在 Coding/Conversation 场景分别平均提升 11%/26%；prefix reuse 计算时间平均提升 22.4%；chunked prefill 最高提升 22.2%；verify attention 即使排除 concatenation 开销仍平均提升 28.6%；decode attention 平均提升 12.9%；SmoothGEMM 平均提升 14.6%；公开端到端 workload achieved QPS 最高提升 89%，TTFT 平均降低 64%，TBT 平均降低 69%；内部 workload 全优化相比 vLLM-APC 提升 95%。

    4、pipeline/kernel 解析：
        - 新 pipeline/kernel 是什么：论文引入两个核心 kernel 级设计：`Meta-Attention` 和 `SmoothGEMM`。Meta-Attention 是一个统一 P/D/V attention 的 meta-kernel，它把不同 stage、不同 mask、不同 KV cache 布局的 attention tile 统一成 GEMM-Softmax-GEMM  primitive，并根据 workload 选择三阶段、四阶段或二阶段 Cube-Vector orchestration pipeline。SmoothGEMM 是面向动态 Linear/GEMM shape 的 kernel 方案，它通过 on-chip virtual padding、selective read/write、固定 tile shape 离线优化和 Task-Table 调度，在不做真实 HBM padding 的情况下保持固定 shape GEMM 的高效率。
        - 新 pipeline/kernel 的执行流例子：一个请求进入 XY-Serve 后，APC 先在 KV cache radix tree / block table 中匹配历史 prefix。未命中的新 token、已有 decode token、speculative verify token 进入 scheduling queues。Scheduler 按固定 budget 形成一个 chunk：例如 chunk 中包含一段 prefill token、若干 decode token 和一棵 speculative verify token tree。系统生成 Token-Table，其中 attention entry 记录每个 stage 的 start、tokenNum、kvLen、tileSize，linear entry 记录 QKV/OProj/GateUp/Down 的 tokenNum 和 nLen。Attention decomposition 将 prefill 按 tileSize 切成多个 tile，将 decode token 切成单 token tile，将 verify token 按 speculative length 和 mask 切成 tile；随后按 tile area 排序并对称 round-robin 分配到 AI cores，形成 Task-Table。Meta-Attention kernel 读取 Task-Table 后，每个 core 取自己的 tile list，从 block table 中读取对应 KV cache，若是 prefix reuse 的 partial block，则通过 copy-on-write 维护 token-wise reuse；若是 tree-based speculative decoding，则只把 specLen * specLen 的 speculative mask 外部传入并逐行加到 attention score，其余 kvLen 区域保持 valid。短序列 mixed workload 走三阶段 pipeline，用 Cube 做 QK/SV、Vector 做 softmax 并重叠执行；长序列因 L2 容量限制走四阶段 pipeline，在 K/V 维分段并加入 update；decode-only workload 走二阶段 pipeline，把 QK 移到 Vector unit 并与 softmax 融合，同时 Cube 做 SV，使 K cache 和 V cache 可并行从 HBM 读取以改善带宽利用率。
        - SmoothGEMM 的执行流例子：同一个 chunk 的 P/D/V token 在 Linear 层被拼接成一个 activation tensor。以 QKV projection 为例，M 是当前 chunk tokenNum，N/K 由模型 hiddenSize、head 数和 TP 决定。若 M 不是 tile size 的倍数，SmoothGEMM 不在 HBM 中真实 padding，而是在 on-chip buffer 以 tile-size 单元分配空间，MTE 从 HBM selective read 真实 token 行，padding 行只在片上虚拟存在；计算仍使用离线调优过的固定 tile GEMM primitive；写回时 selective write 只写真实输出行。对于 Attention GEMM 的 N/K 任意长度，KV cache 的 page/block 已按 tile 对齐读入，softmax 时把 padded 区域置零，并在 SV 阶段自然约去，因此不会污染最终输出。
