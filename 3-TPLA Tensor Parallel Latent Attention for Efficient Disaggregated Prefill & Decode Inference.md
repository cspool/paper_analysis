    1、论文工作：
        - 论文要解决的核心问题：MLA 用低秩 latent vector `cKV` 压缩 KV cache，本来可以降低长上下文 decode 的显存容量和 HBM 读取压力；但在 tensor parallelism 下，多个 GPU 分摊 attention heads 时，每个 GPU 仍然要加载完整的 latent KV cache。这样 MLA 的 KV 压缩收益没有随 TP degree 分摊到每卡内存流量上，decode 阶段仍受 memory bandwidth 限制。论文用 DeepSeek-V3 举例说明，latent KV 维度固定为 `64 + 512 = 576`，在 TP 下每个设备都要复制这份 cache；相比 GQA 可按 head/group 自然切分 KV cache，MLA 在并行部署中失去了 per-device cache 优势。
        - 论文的主要贡献：论文提出 Tensor Parallel Latent Attention（TPLA），在 decode 阶段同时切分 latent representation 和每个 attention head 的输入维度，使每个 GPU 只读取本地 latent KV shard；每个 shard 独立计算局部 attention output，再通过 AllReduce 汇总。与 GLA 不同，TPLA 让每个 attention head 仍覆盖完整 latent space，因此比“每个 head 只看一半 latent”的 GLA 保留更强表达能力。为了直接继承 MLA checkpoint，论文还推导了 RMSNorm 和 softmax 的正交重参数化方法，使用 Hadamard / PCA 变换降低切片带来的跨 shard 误差，并提出 prefill-decode separation：prefill 阶段保留 MLA 形式以保持 compute-bound 阶段效率和精度，decode 阶段切换到 TPLA 以降低每卡 KV bandwidth。
        - 论文所处背景：该工作位于 LLM 推理系统、KV cache 压缩、tensor parallel attention 和 prefill/decode 分离部署的交叉点，主要面向 DeepSeek-V2/V3、Kimi-K2 等 MLA 模型在长上下文、多 GPU TP decode 下的瓶颈。它不是传统 request scheduler，也不是 KV cache 量化/剪枝，而是 attention 表达形式和 TP 切分方式的系统化改造，目标是在不从头训练模型的前提下，把 MLA 的压缩收益真正变成每卡 HBM 流量收益。

    2、相对 Baseline 解决的问题与设计方法：
        - Baseline 的具体问题：MLA baseline 的问题是 latent KV cache 在 TP decode 时被每卡完整复制，per-device memory traffic 没有随 TP 切分下降。GQA baseline 可以按 KV group 切分 cache，但论文认为其表达能力弱于 MLA。GLA baseline 试图把 latent KV 切成组，让每组 heads 只访问本地 latent slice，从而降低每卡 cache；但这种设计丢掉了 off-diagonal query-latent interaction，也就是每个 head 不能看到完整 latent representation。论文实验显示，把 DeepSeek-V2-Lite 的 MLA 直接转成 GLA 会让 WikiText-2 PPL 从 `6.31` 恶化到 `2212`，commonsense 平均准确率从 `61.75` 降到 `33.77`，说明这种 baseline 的精度代价不可接受。
        - 论文的设计方法：TPLA 保留 GLA 的硬件友好思路，即把 latent KV 分布到不同 GPU，但改变 query/head 的组织方式。以 TP=2 为例，TPLA 将 `cKV` 切为两个 latent shard，每个设备上的 attention 使用所有 query heads，只是每个 head 的输入 feature 维度被切片；设备各自计算局部 `O0`、`O1`，最后 AllReduce 得到完整输出。由于 RMSNorm 和 softmax 原本依赖全局 latent vector，TPLA 在切片前引入正交矩阵 `U` 做权重重参数化：Hadamard 变换偏向均衡各 shard 的数值范数，PCA 变换根据 calibration set 的 latent cache 协方差重排维度，通常更适合降低 softmax 切片误差。prefill-decode separation 则在 prefill 期间使用重参数化后的 MLA，不切分 RMSNorm/softmax；decode 时切换到 TPLA，复用 prefill 生成的 KV cache shard。
        - 方法如何对冲 Baseline 缺陷：相对 MLA，TPLA 让每卡只读一部分 latent KV cache，直接降低 decode 阶段最关键的 HBM traffic；相对 GLA，TPLA 不让每个 head 只看局部 latent，而是通过 full-head visibility 保留 MLA 的表达能力；相对从头训练 GLA，TPLA 可以直接继承 DeepSeek / Kimi 这类 MLA checkpoint，只需重参数化和可选轻量 alignment；相对 KV 剪枝、合并或量化，TPLA 不改变 KV 内容本身，而是改变 TP 部署中 KV 的持有和读取路径，因此更少引入信息损失。prefill-decode separation 进一步把结构选择与阶段特征对齐：compute-bound prefill 用 MLA，memory-bound decode 用 TPLA。
        - 关键 trade-off：TPLA 的核心 trade-off 在精度近似、通信、计算复制和适用范围之间。RMSNorm / softmax 的局部切片并不严格等价于全局计算，长上下文和长输出会累积误差；PCA 对 TP=2 的 partition 效果较好，但可能把信息集中到前几个主成分，论文也指出当 group partition `g > 2` 时可能难以保持精度；Hadamard 更利于多组均衡，但对 softmax 的指数敏感性帮助不足。TPLA 降低每卡 KV bandwidth，但需要 AllReduce 汇总输出，并因 head 组织方式带来额外实现复杂度。若从头训练 TPLA，head 数增加可能提高训练计算量；若只做 checkpoint conversion，则仍可能需要 alignment 或 post-pretraining 修复小幅精度损失。

    3、论文实现：
        - Baseline 如何实现：论文主要比较 MLA、GLA、TPLA、TPLA (align) 和 TPLA (pd sep.)。MLA 是 DeepSeek 系列和 Kimi-K2 的原始 attention 形态；GLA 按论文 Section 3.3 的方式将 attention heads 分成两组，每组只访问一半 latent dimension；TPLA 用 PCA-based reparameterization 将 MLA components 转为 TP-friendly 形式；TPLA (align) 在转换后用 SmolLM-Corpus 做轻量 feature / output alignment；TPLA (pd sep.) 在 prefill 阶段保持 MLA 计算，在 decode 阶段使用 TPLA，以避免大部分 token 的 RMSNorm/softmax slicing 误差。
        - 新设计如何实现：TPLA 对 MLA 的 absorbed attention 做代数重写，将 latent KV cache 沿 feature axis 分片，同时切分每个 head 的输入维度，并在设备间 AllReduce 局部输出。重参数化部分先收集 calibration data 的 latent cache 特征；PCA 方法对协方差矩阵做特征分解得到正交矩阵 `U` 和 eigenvalues；Hadamard 方法使用 Walsh-Hadamard 矩阵均衡数值分布。对于 TPLA (align)，论文使用 256 个长度为 2048 的随机样本做 layer-wise input/output MSE 对齐，训练 10 epochs，Muon optimizer，初始学习率 `1e-6`；随后用 100M tokens 做 end-to-end output alignment，batch size 32、learning rate `2e-5`、warmup ratio 0.03、cosine scheduler、max sequence length 4096。速度评估中，模型转为 BF16，使用 FlashAttention-3，并移除 MoE routing 以隔离 attention 速度影响。
        - 实验 / 实现平台：精度实验覆盖 DeepSeek-V2-Lite、DeepSeek-V2、DeepSeek-V3、Kimi-K2-Base、LLaMA-2-7B + TransMLA 桥接模型；短文本任务用 LightEval 评估 WikiText-2 PPL、MMLU、ARC、PIQA、HellaSwag、OpenBookQA、WinoGrande；长上下文任务用 LongBench，DeepSeek-V2-Lite 最大输入 31,500 tokens，DeepSeek-V3 最大输入 127,500 tokens。alignment 在 8x H20 GPU 节点上进行，每卡 96GB，约 148 FP16 TFLOPS。吞吐和 TTFT 实验在两张 H800 GPU 上测 DeepSeek-V3-0324（685B）和 Kimi-K2-Base（1T），并使用不同 context / prefilling length。
        - 关键实验设置与指标：短文本上，DeepSeek-V2-Lite 原始 MLA 的 PPL / 平均准确率为 `6.31 / 61.75`，GLA 退化为 `2212 / 33.77`，TPLA 为 `7.24 / 54.33`，TPLA (align) 恢复到 `6.51 / 61.52`，TPLA (pd sep.) 为 `6.31 / 61.44`，几乎保持原始水平。DeepSeek-V3 原始 commonsense 平均准确率为 `72.10`，TPLA 为 `68.00`；Kimi-K2-Base 原始为 `73.52`，TPLA 为 `70.49`。LongBench 上，DeepSeek-V3 原始平均 `58.19`，TPLA (pd sep.) 为 `56.04`，下降 `2.15`；Kimi-K2-Base 原始 `54.78`，TPLA (pd sep.) 为 `52.39`。吞吐上，在 32K context length，DeepSeek-V3 TPLA decode 比 MLA 快 `1.79x`，Kimi-K2-Base 快 `1.93x`；在 4096 decode length 设置下，论文概括 TPLA 可接近 `2x` throughput。prefill 上，TPLA (pd sep.) 相比直接 TPLA 在 1K prompt length 约快 `1.4x`，结论部分概括 PD separation 最高可带来 `29%` TTFT latency reduction。

    4、pipeline/kernel解析：
        - 新pipeline/kernel是什么：论文没有给出一个单独命名的新 CUDA kernel；最接近的新执行路径是 “prefill 用 MLA、decode 用 TPLA 的 tensor-parallel latent attention pipeline”。它由三段组成：第一，prefill 阶段执行重参数化后的 MLA，不切分 RMSNorm/softmax，以减少计算复制和近似误差；第二，decode 阶段每张 GPU 只持有并读取本地 latent KV shard，每个 attention head 的输入维度也被切到对应设备；第三，每卡独立运行 attention 计算后，通过 AllReduce 把局部 output 合成完整 hidden output。论文说明 TPLA 可建立在 FlashAttention-3 之上实现 practical end-to-end acceleration，但没有公开给出完整 kernel 代码。
        - 新pipeline/kernel的执行流例子：以 TP=2、单个 decode token 为例，prefill 时请求的长 prompt 先按 MLA 形式生成 latent KV cache；经过正交重参数化后，这份 cache 可以被拆成 `cKV0` 和 `cKV1` 两个 shard。decode 新 token 到来时，GPU0 读取 `cKV0`，GPU1 读取 `cKV1`；两张卡都拥有完整 query-head visibility，但每张卡只计算对应 feature slice 上的 `Q0` 或 `Q1` 与本地 `cKV` 的 attention。GPU0 得到局部输出 `O0 * W0VO`，GPU1 得到 `O1 * W1VO`；随后两张卡执行 AllReduce，把局部输出求和成最终 attention output，再进入后续 FFN / MoE 等层。这样，一个 token 的 decode 路径从“每卡读完整 576-d latent cache”变为“每卡读本地约一半 latent cache并汇总输出”，decode 的瓶颈从 HBM 读取压力中释放一部分，但代价是 AllReduce 和局部近似误差。
