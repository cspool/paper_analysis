1、论文工作：
    - 论文要解决的核心问题：低比特 LLM 推理中，rotation-based quantization、fine-grained group quantization 和 asymmetric quantization 各自能提升量化精度，但直接组合会产生算法和硬件两层冲突。算法上，rotation 倾向于把 outlier 全局打散，group quantization 则依赖小 group 内的局部分布和 scale/zero-point 捕捉局部偏移；当 group size 变细，例如 32、16 或更小，rotation 反而可能破坏 group 的 scale coherence，使 perplexity 变差。硬件上，小 group 和 asymmetric quantization 需要频繁处理 per-group scale factor 与 zero-point，既增加 metadata，也把低比特 INT GEMM 后的反量化推向浮点 datapath，削弱 INT4 推理的面积、能耗和吞吐收益。
    - 论文的主要贡献：GyRot 提出一个 algorithm-hardware co-design。算法层提出 CoRFiG，即 Coarse Rotation, Fine Grouping，把 rotation scope R 和 quantization group size G 解耦，让 R = 2^g * G，在较粗的局部范围内旋转，而继续使用较细的 group quantization；提出 HAP，即 Harmonic-Aligned Permutation，把高幅值 outlier channel 映射到 Hadamard matrix 的 harmonic rows，使同一 group 内 outlier 以更一致的符号参与旋转，从而收窄 group-wise range。量化层重写 asymmetric quantization 的顺序，把 zero-point 从 scaled domain 改到 unscaled domain，并用 ceiling-based zero-point rounding 避免下界 underflow。硬件层实现 GyRot accelerator：8 x 8 systolic PE array，每个 PE 做 32-way INT4 dot product，并集成全整数反量化路径，scale factor 和 zero-point 均用 INT8 表示。
    - 论文所处背景：LLM 低比特推理常见瓶颈不只是矩阵乘的 bitwidth，还包括 activation/weight outlier、KV cache、per-group metadata 和 dequantization。Quarot、SpinQuant、DuQuant 等 rotation 方法用 Hadamard 或可训练 rotation 平滑 outlier；MANT、Tender、LightRot 等硬件或算法方案用 group quantization、dynamic scale 或 rotation 支持低比特推理。但论文指出，rotation 尚未和 fine-grained group quantization 自然协同，尤其当目标是 W4A4KV4、group size 32、scale/zero-point 低精度且硬件全整数执行时，现有方法要么精度掉得多，要么依赖 FP16/FP scale 和 zero-point 带来额外硬件成本。

2、相对 Baseline 解决的问题与设计方法：
    - Baseline 的具体问题：第一类 baseline 是全局或强 outlier-dispersing rotation，如 Quarot、SpinQuant、DuQuant。它们改善 per-channel 或大 group 量化，但当 group size 缩小后，group quantization 已经能局部吸收 outlier，继续全局打散 outlier 会让每个小 group 的分布更不稳定，论文在 LLaMA3-8B/WikiText-2 上展示了 rotation 与小 group 组合可能不再改善，甚至在 RTN 下反转为更差 perplexity。第二类 baseline 是 fine-grained group quantization 加 asymmetric quantization，例如 AMXFP 和 LightRot 相关思路。它们利用 per-group scale/zero-point 提升精度，但小 group 会显著增加 dequantization 次数；如果 scale factor 或 zero-point 仍需 FP16/FP datapath，则 INT4 GEMM 的能效优势被反量化逻辑侵蚀。第三类硬件 baseline 包括 Tender、MANT、LightRot accelerator。Tender 的 W4A4 精度下降明显；MANT 使用 flexible number format 和 group size 64；LightRot 以 R=G=128 的局部 rotation 与 outlier alignment 支持 asymmetric group quantization，但依赖浮点 zero-point，不易扩展到 G=64/32 的全整数低开销路径。
    - 论文的设计方法：GyRot 的关键不是单独提出一个新量化格式，而是控制 rotation 和 group quantization 的相互作用。CoRFiG 使用局部但较粗的 rotation scope R，例如论文默认 R=1024、G=32，使 outlier 只在 R 个 channel 内被平滑，避免全局扩散，同时保留 32-channel fine grouping 的局部 scale 优势。HAP 利用 Hadamard 矩阵递归结构中的 harmonic rows，把高幅值 channel 预先 permutation 到这些行，使旋转后每个 group 的 outlier 符号模式更稳定，group-wise range 更窄。随后，论文把 conventional asymmetric quantization 中的 x/sx + zx 改写为 (x + zx)/sx，其中 zx = ceil(-min(xg))，这样 zero-point 不再被很小的 sx 放大成长尾分布，并且 ceiling 让量化后的 zQ 不小于真实 z，避免 xmin + zQ 下溢导致 clipping。
    - 方法如何对冲 Baseline 缺陷：对 Quarot/SpinQuant/DuQuant 的问题，GyRot 不把 outlier 无约束地全局打散，而是通过 R=2^g*G 把 rotation 限在与 group 结构相容的局部窗口内；对 LightRot 的 R=G 限制，GyRot 允许 rotation 比 group 更粗，从而既保留 flattening，又避免每个 group 自己旋转导致的平滑不足。对 asymmetric group quantization 的硬件开销，GyRot 通过 HAP 降低 scale factor 精度敏感性，通过重写 zero-point 公式和 ceiling rounding 降低 zero-point 精度敏感性，最终让 SF/ZP 可用 INT8 表示。对硬件 baseline 的 FP dequantization 问题，GyRot 在 PE 内把 INT4 dot product、INT8 scale/zero-point、WSUM correction、inter-group accumulation 做成整数 pipeline，减少浮点转换和浮点反量化逻辑。
    - 关键 trade-off：GyRot 接受了更复杂的离线 permutation/rotation-aware quantization 流程和专用硬件 datapath，换取 W4A4 细粒度 group quantization 下的精度和能效。CoRFiG 的 R 不能过小，否则 rotation 平滑不足，论文中 G32/R32、R64 的 PPL 很差；R 过大或 global rotation 又会削弱 group locality，因此论文选 G32/R1024 作为默认点。HAP 会让 group-wise distribution 更偏斜，所以必须搭配 reformulated asymmetric quantization 和 zero-point ceiling。硬件上，GyRot 需要专用 PE、FHT unit、多 bank buffer 和 metadata path；这不等同于可直接在现有 GPU Tensor Core 上复用同样收益。论文也承认 GPU 上 group quantized inference 仍会因为 CUDA core 上的 FP dequantization 而难以完全发挥整数算术优势，因此它选择专用 accelerator 作为实现目标。

3、论文实现：
    - Baseline 如何实现：算法精度对比包含 Tender、MANT、Quarot、SpinQuant、DuQuant、LightRot 等。Tender 和 MANT 被作为硬件友好 quantization baseline，其中 Tender 评估 W8A8 和 W4A4，MANT 评估 W4A8 和 W4A4/G64；rotation-based baseline 在 W4A4KV4 下比较 Quarot、SpinQuant、DuQuant 和 LightRot。论文为公平比较设定动态 asymmetric per-token activation quantization，KV cache 采用 group size 128 的 asymmetric quantization，weights 在 rotation 后用 GPTQ 做 symmetric quantization。硬件评测中，Tender、MANT、LightRot 被建模为带对应 dequantization unit 的 systolic PE array，并在 iso-compute-area 约束下比较。
    - 新设计如何实现：GyRot-INT 默认配置为 activation 和 weight 都使用 4-bit、group size 32，KV cache 使用 4-bit、group size 128；scale factor 和 zero-point 使用 INT8。PE 每周期做 32-way INT4 activation-weight dot product，产生 13-bit partial sum。反量化阶段先应用 activation scaling factor SX，同时把 zero-point ZX 与预计算的 group-wise weight sum WSUM 相乘并加入修正项，再应用 weight scaling factor SW；计算保持整数域，inter-group accumulation 使用 32-bit integer accumulator，最终写回前转换为 FP16 以降低输出位宽。加速器采用 8 x 8 systolic PE array，每个 PE 支持 32-way INT4 dot product，因此 8 x 8 x 32 tensor array 可并行执行 2048 operations。输入 buffer 存 X/SX/ZX，weight buffer 存 W/SW，WSUM 每行共享，多 bank memory 为数据和 metadata 提供带宽。
    - 在线 rotation / FHT 实现：论文指出多数 rotation 可基于矩阵乘 rotation-invariance 离线融合进前后层 weights，但 embedding、activation/self-gated activation 等非线性或 element-wise operation 之间仍需要 online rotation。GyRot 在 Fused Vector Unit 中集成 Hadamard rotation 和 quantization unit。FHT unit 使用 5-stage、32-way 结构，共 160 个 add/subtract units，每 stage 32 个；通过 local register file 和 two-stage scheme 支持最大 32 x 32 = 1024 维 rotation scope，并用 partial gating 支持 2、4、8、16、32 等 power-of-two 子尺寸，避免小 R 浪费能量。HAP 的 permutation 可预融合到 weight output channels，因此论文称它没有额外 runtime overhead。
    - 实验 / 实现平台：算法评测覆盖 LLaMA、LLaMA-2、LLaMA-3 系列。语言建模精度使用 WikiText-2 perplexity；zero-shot task 使用 PIQA、ARC-e、ARC-c、BoolQ、HellaSwag、WinoGrande，并通过 LM-Evaluation-Harness 评估；对话质量使用 MT-Bench 和 LLM-as-a-Judge，模型包括 LLaMA-3-8B-Instruct。硬件实现方面，PE 和相关模块用 SystemVerilog RTL 实现，并通过 RTL simulation 验证功能；综合使用 Samsung 28nm technology node 和 Synopsys Design Compiler，目标频率 1GHz；片上 SRAM 用 commercial memory compiler 生成；DRAM power 用 Micron DRAM Power Calculator 的 DDR4 model 估算。
    - 关键实验设置与指标：精度方面，GyRot-INT 在 W4A4KV4 下，LLaMA-1/2 的 perplexity 接近 GyRot-FP，并优于或接近 Quarot、SpinQuant、DuQuant、MANT、LightRot 等；LLaMA-3-8B zero-shot 平均准确率中，Full Precision 为 74.19，Quarot 为 66.96，SpinQuant 为 69.90，LightRot 为 72.43，GyRot-INT 为 72.98，论文称相对 full precision 仅 1.2% drop。MT-Bench 上，GyRot 相对 Quarot、SpinQuant、LightRot 的 adjusted win rate 分别为 66.6%、68.8%、54.7%。设计空间中，G32/R1024 在 LLaMA-3-8B 上 PPL 为 6.91，优于 global rotation 的 7.04；SF 精度实验显示 GQ-only 在 INT8 SF 下 PPL 崩到 416.15，而 CoRFiG+HAP 在 INT8 SF 下保持 6.80；ZP 实验中，CoRFiG+HAP 加 reformulated formula 和 ceiling rounding 后，INT8 ZP PPL 为 6.91，接近 FP16 的 6.81。硬件方面，GyRot-INT 相对 Tender 达到 geometric mean 3.40x speedup 和 3.64x energy efficiency；相对 MANT/LightRot 等 group quantization baseline，平均 speedup 高 41.7%-67.5%，energy efficiency 高 19.8%-31.4%。PE 级别，GyRot-INT 相比 Tender 取得 65.2% area reduction 和 69.2% energy reduction；总 accelerator 面积 2.10 mm2、功耗 740.95 mW，其中 global buffer 占 57.1% 面积，INT tensor PE 占 55.4% 功耗，FVU/FHT 只占 3.5% 面积和 0.9% 功耗。

4、pipeline/kernel 解析：
    - 新 pipeline/kernel 是什么：GyRot 的核心执行路径可以概括为“CoRFiG+HAP 量化布局 + 全整数 group-wise asymmetric dequantization PE + 在线 FHT rotation FVU”。它不是 GPU kernel，而是专用 LLM accelerator 中的 PE/FVU pipeline。离线阶段，模型 activation/weight 的量化流程先选择 group size G 和 rotation scope R；CoRFiG 在每个 R 范围内应用 Hadamard rotation；HAP 根据 Hadamard harmonic rows 对 outlier channels 做 permutation；weights 中可融合的 inverse rotation/permutation 被预融合。在线阶段，线性层使用 INT4 activation 和 INT4 weight 做 32-way dot product，PE 内直接完成 INT8 scale/zero-point 反量化和 inter-group accumulation；遇到 embedding、activation 等无法离线融合 rotation 的位置，则 FVU 用 FHT 执行在线 rotation，再进行量化并送往后续矩阵乘。
    - 新 pipeline/kernel 的执行流例子：假设某个 decoder block 的线性投影要计算 Y = XW，GyRot 先在离线量化时把 channel 以 G=32 分组，并选择 R=1024 的 coarse local rotation。HAP 找出高幅值 outlier channels，把它们排列到 R 范围内的 harmonic rows，使旋转后每个 32-channel group 的分布更紧。部署时，weight buffer 存放 INT4 weights 和 INT8 SW，input buffer 存放 INT4 activations 及 INT8 SX/ZX。运行时，8 x 8 systolic array 中的每个 PE 每周期读取 32 个 INT4 X 和 32 个 INT4 W，完成 32-way dot product 得到 partial sum。随后 PE 在整数 datapath 中先把 SX 作用到 partial sum；并行地，用当前 group 的 ZX 乘以该 group 预计算 WSUM，形成 asymmetric zero-point correction；之后再乘 SW，得到当前 group 的 dequantized contribution。多个 group 的 contribution 在 32-bit integer accumulator 中顺序累加，最终输出前转为 FP16。
    - 在线 rotation 路径例子：如果某层之间存在非线性操作，rotation 不能完全吸收到相邻 weight。此时 FVU 从 global memory 读取 activation，先执行 nonlinear/element-wise operation，再把向量送入 FHT rotation unit。以 R=1024 为例，FHT unit 用 32-way lanes 分两阶段处理 32 x 32 维 rotation；每个 stage 执行 Hadamard butterfly 形式的 add/subtract，partial gating 允许较小 power-of-two R 关闭无用 lane。旋转后的 activation 保持 CoRFiG/HAP 对 group locality 的设计假设，随后被 asymmetric group quantization 成 INT4 activation 及 INT8 SX/ZX metadata，送入 PE array 做后续矩阵乘。
    - pipeline 的边界和限制：GyRot 最适合 W4A4KV4 或 W3A4 这类低比特、group quantization 较细且硬件可集成整数 metadata path 的 LLM inference。它的实验结论依赖专用 RTL accelerator、28nm 综合、1GHz 目标频率和 DDR4 power model，不等同于现成 GPU 上的端到端速度。论文没有公开代码链接，也没有提供可复现实验 artifact，因此具体 RTL、量化脚本、调度器、memory timing 和 compiler/runtime 接口无法从公开仓库验证。算法方面，R/G 选择、HAP permutation、SF/ZP 精度和模型类型会影响效果；如果目标硬件没有高效整数 dequantization path，或者工作负载的 online rotation 比例过高，GyRot 的硬件收益可能会被额外数据搬运、metadata 访问或 FHT 延迟抵消。
